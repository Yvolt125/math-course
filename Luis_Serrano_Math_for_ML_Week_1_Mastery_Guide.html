1. Data as Geometric Objects Scalars · Vectors · Matrices

To understand Linear Algebra, we must first stop seeing numbers as just symbols and start seeing them as coordinates in space. Every piece of data has a physical position. This shift in perspective — from numbers to geometry — is the single most important idea in this guide.

The Scalar — A Single Value

A Scalar is a single number. It represents magnitude (how much) but has no direction. It is a point on a 1D number line.

Example: The temperature in a room is 22°C. It's one value that tells you "how hot" it is, but it doesn't point anywhere.

The Vector — A Data Point in Space

A Vector is an ordered list of scalars. In ML, we call this a Feature Vector. Geometrically, a vector is an arrow pointing from the origin \((0,0)\) to a specific coordinate. Stacking multiple scalars into a vector is how we describe a data point with more than one property at once.

Example: A house described by [2100 sqft, 3 bedrooms] becomes: \[\vec{v} = \begin{bmatrix} 2100 \\ 3 \end{bmatrix}\] It lives in a 2D space where the horizontal axis is "Size" and the vertical axis is "Rooms." Every house in our dataset is a unique arrow pointing to its own location in this space.

The Matrix — A Complete Dataset

A Matrix is a collection of vectors arranged as rows. If one vector describes one house, a matrix describes an entire neighborhood — your whole dataset packed into a single rectangular grid.

\[ A = \begin{bmatrix} 2100 & 3 \\ 1500 & 2 \\ 3000 & 4 \end{bmatrix} \leftarrow \text{3 houses (rows) × 2 features (columns)} \]

The hierarchy is simple: a scalar is a single value, a vector is a column of scalars, and a matrix is a grid of vectors. Each level adds one more dimension of structure.

Where we're headed: We now know that data lives as points and arrows in geometric space. The natural next question is — what rules connect those points to outcomes? How does one feature (like house size) relate to a prediction (like price)? That relationship is a Linear Equation.

2. The Linear Equation Modeling relationships between features and outcomes

A linear equation is the mathematical way of saying: "these features combine, in a straight-line fashion, to produce this outcome." This is the core operation of nearly all Machine Learning models.

From High School to ML:

In school, we learn a line as \(y = mx + b\). In Machine Learning, the same equation scales to many features at once:

  • \(y\) — Label / Prediction: The output we want to estimate (e.g., price of a house).
  • \(x_i\) — Features: The inputs we already know (e.g., size, number of rooms).
  • \(w_i\) — Weights: The importance given to each feature — the "slopes." Does room count matter more than square footage? The weights quantify this. Finding the right weights is literally what Machine Learning is doing.
  • \(b\) — Bias: The "y-intercept." It shifts the model's starting point away from the origin, so it isn't forced to go through \((0,0)\) — which would be unrealistic for most real data.
\[ y = w_1 x_1 + w_2 x_2 + \cdots + b \]
Why "Linear"?

We only use addition and scaling — no squaring (\(x^2\)), no exponents. Geometrically, this means the relationship is always a straight line in 2D or a flat plane in 3D. There are no curves. This constraint makes the math tractable and, as we'll see, leads naturally to the language of matrices and systems of equations.

Where we're headed: One linear equation is one rule — one line or plane. But in practice we have many data points, each imposing its own constraint. When we stack multiple equations together, we get a System of Equations, and our goal becomes finding the single point that satisfies all of them simultaneously.

3. Systems of Equations Many rules, one answer — encoding constraints as a matrix

In practice, we don't have one house — we have hundreds. Each house gives us one equation, one constraint. A System of Equations is the full collection of these constraints working together. Solving the system means finding the single point in space that satisfies every constraint simultaneously. Geometrically, this is the point where all lines (or planes) intersect.

Concrete Example (2D):
Constraint A: \(x + y = 5\)   (sum is 5)
Constraint B: \(x - y = 1\)   (difference is 1)
Solution: Only the point \((3, 2)\) satisfies both at once. Every other point satisfies at most one of them. This is what "solving the system" means.
Why the unique solution is the whole point — in ML terms:

In a Machine Learning context, the variables we are solving for are the model's weights \((w_1, w_2, \ldots, b)\). Each data point we have observed is one equation — one constraint that says "given these feature values, the output must be this." Solving the system means finding the one set of weights that is consistent with every data point at once.

A unique solution is the ideal outcome: it means the data has fully determined the model. There is exactly one set of weights that fits all the evidence — a single, unambiguous answer. This is what we hand to the prediction step.

Without a unique solution, the system breaks down in one of two ways, and neither is usable:

  • Infinite solutions (redundant data): Many different sets of weights fit the data equally well. The model is underdetermined — there is no principled reason to choose one over another. Any prediction we make is arbitrary, because we never actually identified which rule the data follows.
  • No solution (contradictory data): No set of weights can satisfy all the constraints at once. The data is mathematically inconsistent — it is as if some of your observations contradict others. A linear model simply cannot exist for this data in this form.

In short: a unique solution means "we found the rule." Infinite solutions means "we can't tell which rule it is." No solution means "no such rule exists."

The Augmented Matrix — Encoding a System Compactly

Writing out full equations becomes unwieldy with many variables. We encode the system into a rectangular grid: the Augmented Matrix. The coefficients sit on the left of a dividing bar, the right-hand-side results sit on the right.

For the system \(2x + y = 5\) and \(x - y = 1\): \[ \text{Coefficient Matrix: } A = \begin{bmatrix} 2 & 1 \\ 1 & -1 \end{bmatrix} \qquad \text{Augmented Matrix: } \left[ \begin{array}{cc|c} 2 & 1 & 5 \\ 1 & -1 & 1 \end{array} \right] \] Every row is one equation. Every column is one variable. The bar separates "the rules" from "the targets." This compact form is what Gaussian Elimination (Section 6) will operate on.

Extending to 3D — Planes Instead of Lines

In 3D, each equation defines a plane. The solution is still the single point where all planes meet.

Three planes, three constraints: \[ \begin{cases} z = 3 & \text{(horizontal ground plane)} \\ y = 1 & \text{(vertical wall)} \\ x + y + z = 6 & \text{(diagonal sheet)} \end{cases} \] Rotate the interactive plot below to see all three planes. They meet at exactly one point: (2, 1, 3).
Where we're headed: The examples above were "clean" — lines that crossed neatly, planes that converged. But what if our equations repeat the same information, or contradict each other? Before we can solve a system, we need to understand when and why it fails — which comes down to the concept of Linear Independence.

4. Linear Independence, Rank and Singularity Measuring real information — when systems fail and why

For a system to have a unique solution, each equation must bring genuinely new information — it must cut through space in a direction no other equation already covers. The formal name for this property is Linear Independence. When it breaks down, the system becomes Singular and cannot be uniquely solved.

Linear Independence vs. Dependence

Think of each equation as pointing in a geometric direction. The question is whether your equations collectively point in enough distinct directions to pin down a unique answer.

  • Linearly Independent: Each equation cuts through space in a genuinely new direction, narrowing the solution further. Together they converge on one unique intersection point.
  • Linearly Dependent: At least one equation can be produced by scaling or combining the others. It adds zero new geometric direction and zero new information — and the system collapses.

Rank — Counting the Real Information

Rank is the formal measure of how many equations in a system are genuinely independent. It tells you the true dimensionality of your data — which may be lower than it appears on the surface.

  • Full Rank (e.g., Rank 3 in a 3×3 system): All equations are independent. You have genuine 3D data — enough constraints to pinpoint a unique solution.
  • Rank 2 (one redundant equation): Your "3D" system is actually flat — the data lives on a 2D plane, and you cannot pinpoint a unique solution.
  • Rank 1 (only one unique equation): Everything collapses to a single line. All your other equations were just scaled copies of the first.
The Shadow Analogy:

Think of Rank as the dimensionality of a shadow. A 3D object always casts a 3D presence in space — but depending on the light angle, its shadow on the wall might be 2D (Rank 2) or even 1D, a thin line (Rank 1). The shadow has lost depth. Rank tells you how much genuine "depth" — how many truly distinct directions — survives in your matrix, versus how much has collapsed into redundancy.

How Singularity Manifests — Two Failure Modes

Linear dependence causes systems to fail in two different ways, depending on how the equations relate to each other:

Failure Mode 1 — Dependent Equations (Infinite Solutions)

When one equation is just a scaled copy of another, the two geometric objects (lines or planes) are identical — they lie exactly on top of each other. There is no single intersection; instead, infinitely many points satisfy both constraints.

Rule 1: "House price is $200 per sqft."   →   \(p = 200s\)
Rule 2: "House price is $400 per 2 sqft."   →   \(p = 200s\)   (same rule, scaled ×2)
Result: Rule 2 is linearly dependent on Rule 1 — it contributes nothing new. Any house satisfying the first automatically satisfies the second. The system has infinitely many solutions because you effectively have only one constraint, not two.

Failure Mode 2 — Contradictory Equations (No Solution)

When two equations share the same left-hand side (same direction in space) but demand different right-hand-side values, they are parallel — they never intersect.

Rule 1: \(x + y = 10\)
Rule 2: \(x + y = 20\)
Result: The left-hand sides are identical (both equations point in exactly the same geometric direction), yet they contradict each other. No pair of numbers can sum to both 10 and 20. Geometrically, these are parallel lines — same slope, different intercepts — that never touch. There is no solution.
ML Warning — Redundant Features: If you include both "Temperature in Celsius" and "Temperature in Fahrenheit" as features in your model, you have created perfect linear dependence — one is always \(F = 1.8C + 32\). The data matrix loses rank, becomes singular, and matrix operations will either crash or produce wildly unstable results. Always check your features for hidden linear relationships before training. Techniques like PCA (Principal Component Analysis) are fundamentally about finding and discarding such redundant dimensions.

Summary — The Three Possible Outcomes

Outcome Root Cause Geometry Matrix Property
One Unique Solution All equations independent Lines / planes intersect at one point Non-singular · Full Rank
Infinite Solutions Dependent — one equation repeats another Lines / planes overlap entirely Singular · Rank-deficient
No Solution Same direction, contradictory values Parallel lines / planes that never meet Singular · Inconsistent
Where we're headed: Rank tells us how many independent dimensions a matrix has, but computing it requires examining the whole matrix row by row. We need a faster, single-number test that tells us immediately — without that inspection — whether a system is solvable. That number is the Determinant.

5. The Determinant A single number that tests whether a system is solvable

The Determinant (\(\det\)) is a single number computed from a matrix that summarises what the matrix transformation does to space — specifically, how much it stretches or squashes the volume of space. This makes it a powerful one-number diagnostic for singularity.

Visualizing the Determinant:
Imagine a unit square at the origin, with area = 1. Apply a matrix transformation:
  • The square stretches into a parallelogram with area 6 → \(\det = 6\)
  • The square flips (mirror image) with area 6 → \(\det = -6\) (sign encodes orientation)
  • The square squashes flat into a line, area = 0 → \(\det = 0\) — the critical case
When \(\det = 0\), the transformation has permanently destroyed a dimension. That lost information can never be recovered.

Calculation — 2×2 Matrix

For \(A = \begin{bmatrix} a & b \\ c & d \end{bmatrix}\):

\[ \det(A) = ad - bc \]

Multiply the main diagonal \((a \times d)\), subtract the off-diagonal \((b \times c)\). The result is the signed area of the parallelogram formed by the matrix's two column vectors.

Calculation — 3×3 Matrix (Rule of Sarrus)

For \(A = \begin{bmatrix} a & b & c \\ d & e & f \\ g & h & i \end{bmatrix}\):

\[ \det(A) = (aei + bfg + cdh) - (ceg + afh + bdi) \]

Add the three "down-right" diagonal products, subtract the three "down-left" diagonal products. The result is the signed volume of the parallelepiped formed by the three column vectors.

The Critical Connection:

A matrix is singular (no unique solution, non-invertible, rank-deficient) if and only if its determinant equals zero. Singularity, Rank, and the Determinant are three different ways of describing the same underlying collapse:

  • Geometrically: the transformation squashes space into a lower dimension
  • In terms of information: at least one equation is linearly dependent (Rank drops)
  • Numerically: the determinant hits zero
Where we're headed: We now know when a system can be solved (det ≠ 0 → full rank → non-singular). The next question is: how do we actually find the solution? We need a step-by-step algorithm. That algorithm is Gaussian Elimination.

6. Gaussian Elimination A step-by-step algorithm for solving systems of equations

Once we know a system is solvable (det ≠ 0, full rank), we need a procedure to actually find the answer. Gaussian Elimination is that procedure — a systematic algorithm that simplifies the augmented matrix step by step until the solutions are immediately readable.

The Target: Reduced Row Echelon Form (RREF)

The goal is to transform the augmented matrix until the coefficient side looks like the Identity Matrix — the "number 1 of the matrix world." Just as \(5 \times 1 = 5\), multiplying any matrix by the Identity leaves it unchanged (\(AI = A\)):

\[ I_{2\times2} = \begin{bmatrix} 1 & 0 \\ 0 & 1 \end{bmatrix}, \qquad I_{3\times3} = \begin{bmatrix} 1 & 0 & 0 \\ 0 & 1 & 0 \\ 0 & 0 & 1 \end{bmatrix} \]

When the coefficient side reaches this "clean" identity pattern, the right-hand side of the augmented matrix directly reveals the solution values. This final form is called Reduced Row Echelon Form (RREF).

The Three Legal Moves — Elementary Row Operations

  1. Swap: Exchange any two rows — reordering equations doesn't change the solution.
  2. Scale: Multiply a row by any non-zero number — scaling an equation doesn't change its constraint.
  3. Combine: Add or subtract a multiple of one row from another — combining equations still describes the same intersection point.
Why these are safe: Each move changes the appearance of the rules but never changes the underlying truth — the intersection point remains exactly the same. It's like translating a legal contract into another language. The words change; the agreement does not.

The Pivot — The Anchor of Each Step

A Pivot is the first non-zero entry in a row. At each step of the algorithm, we use the current pivot to eliminate (zero out) all other entries in its column — above and below — marching the matrix progressively closer to the identity pattern.

Step-by-Step Walkthrough: Solve \(x + y = 3\) and \(2x + 4y = 10\)

1 Write the Augmented Matrix. The bold 1 at position (1,1) is our first Pivot: \[\left[ \begin{array}{cc|c} \mathbf{1} & 1 & 3 \\ 2 & 4 & 10 \end{array} \right]\] 2 Eliminate below the pivot using \(R_2 \leftarrow R_2 - 2R_1\). The 2 beneath the pivot becomes 0: \[\left[ \begin{array}{cc|c} 1 & 1 & 3 \\ 0 & \mathbf{2} & 4 \end{array} \right]\] This staircase shape — zeros below each pivot — is Row Echelon Form (REF). We could back-substitute from here, but let's go further.

3 Reach RREF: Scale \(R_2 \leftarrow \frac{1}{2}R_2\) to make the pivot 1, then subtract from \(R_1\) to zero out above the pivot: \[\left[ \begin{array}{cc|c} 1 & 0 & 1 \\ 0 & 1 & 2 \end{array} \right]\] The coefficient side is now the Identity Matrix. The right-hand side reads directly: \(x = 1,\ y = 2\). Done.
REF vs. RREF:
  • REF: Staircase of zeros below each pivot. Good for back-substitution.
  • RREF: Each pivot is 1, and zeros appear both below and above it. Solution is directly visible — no substitution needed.
Where we're headed: Gaussian Elimination solves one specific system for one specific right-hand side \(b\). But what if we want a general tool — one that can answer \(Ax = b\) for any \(b\), by effectively "undoing" whatever the matrix \(A\) does? That is the Matrix Inverse.

7. The Matrix Inverse Undoing a transformation — and why it sometimes can't be done

Every matrix \(A\) performs a transformation: it takes an input vector \(x\) and maps it to an output vector \(b\), i.e., \(Ax = b\). The Inverse Matrix (\(A^{-1}\)) perfectly reverses that transformation — given the output \(b\), it recovers the original input \(x\).

\[ Ax = b \quad \Longrightarrow \quad x = A^{-1}b \]

Think of \(A\) as a function that scrambles an egg. \(A^{-1}\) is the (imaginary) function that unscrambles it. The inverse only exists if the original transformation was lossless — if no information was destroyed in the process.

The Fundamental Constraint — everything converges here:

The inverse of \(A\) exists if and only if \(\det(A) \neq 0\). Here is why all four ideas connect at this one point:

  • If \(\det(A) = 0\), the transformation squashed 3D space into a flat 2D plane.
  • Once a dimension is gone, the information it carried is gone forever — you cannot reconstruct a unique 3D object from a 2D shadow.
  • This is the same event as rank dropping below full rank (linear dependence), which is the same event as the system being singular (no unique solution).

Singularity · Rank deficiency · det = 0 · No inverse — these are four descriptions of a single underlying collapse. Once you see that, they stop feeling like four separate facts and start feeling like one.

In ML — Linear Regression: The closed-form solution for linear regression weights is: \[ w = (X^T X)^{-1} X^T y \] This formula literally requires inverting the matrix \(X^TX\). If your dataset contains redundant features (making \(X^TX\) singular, det = 0), this formula breaks down entirely. That is the mathematical root of why correlated features cause instability in linear models.
Where we're headed: We now have a complete theoretical toolkit. In practice, these operations scale to datasets with millions of rows — far beyond hand calculation. The final step is implementing all of this in code using NumPy.

8. Python with NumPy Applying every concept at production scale

NumPy provides battle-tested, numerically stable implementations of every concept covered in Sections 1–7. The code below walks through a complete system from representation to solution, with each step annotated back to the relevant concept.

Theory vs. Practice — which solver to use?

The theoretical formula is \(x = A^{-1}b\) (Section 7), but in professional code we prefer np.linalg.solve. Internally it uses a numerically stable variant of Gaussian Elimination (Section 6) — faster than computing a full inverse and far less susceptible to floating-point rounding errors on large systems.

import numpy as np

# § 1 — Represent the system as a matrix
# Solving: x + 2y = 5  and  3x + 4y = 11
A = np.array([[1, 2],
              [3, 4]])
b = np.array([5, 11])

# § 5 — Check the Determinant before solving
det = np.linalg.det(A)
print(f"Determinant: {det:.4f}")

if np.isclose(det, 0):
    # det = 0 → singular → rank-deficient → no unique solution (§ 4 & 5)
    print("Cannot solve: matrix is singular.")
else:
    # § 7 — Method A: via Inverse (theoretical)
    x_inv = np.linalg.inv(A) @ b
    print(f"Via inverse:      x = {x_inv}")

    # § 6 — Method B: via Gaussian Elimination (recommended)
    x_solve = np.linalg.solve(A, b)
    print(f"Via linalg.solve: x = {x_solve}")

# § 4 — Confirm the true dimensionality (Rank)
rank = np.linalg.matrix_rank(A)
print(f"Rank: {rank} / {A.shape[0]}  ({'full rank' if rank == A.shape[0] else 'rank-deficient'})")
