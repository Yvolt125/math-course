What does the derivative \(f'(x)\) measure? The instantaneous rate of change of \(f\) at point \(x\) — the slope of the tangent line.
\[ f'(x) = \lim_{h\to 0}\frac{f(x+h)-f(x)}{h} \]
In ML: how much does the loss change per unit change in this weight?
State the Power Rule for differentiation.
\[ \frac{d}{dx}[x^n] = nx^{n-1} \]
Examples: \(\frac{d}{dx}[x^3] = 3x^2\), \(\frac{d}{dx}[x^5] = 5x^4\), \(\frac{d}{dx}[x] = 1\), \(\frac{d}{dx}[c] = 0\).
Give the derivatives of \(e^x\), \(\ln x\), \(\sin x\), and \(\cos x\).
  • \(\frac{d}{dx}[e^x] = e^x\) — the only function that is its own derivative
  • \(\frac{d}{dx}[\ln x] = \frac{1}{x}\)
  • \(\frac{d}{dx}[\sin x] = \cos x\)
  • \(\frac{d}{dx}[\cos x] = -\sin x\)
What is a critical point? How do you distinguish minima from maxima? A critical point is where \(f'(x) = 0\). Use the second derivative test:
  • \(f''(x) > 0\): curves up → local minimum
  • \(f''(x) < 0\): curves down → local maximum
  • \(f''(x) = 0\): inconclusive (could be a saddle)
Gradient descent seeks points where the loss gradient is zero (its minimum).
Geometrically, what does \(f'(x) > 0\) vs \(f'(x) < 0\) tell you?
  • \(f'(x) > 0\): the function is increasing at \(x\) — slope is positive
  • \(f'(x) < 0\): the function is decreasing at \(x\) — slope is negative
  • \(f'(x) = 0\): the function is flat at \(x\) — candidate for min/max
State the sum rule and constant multiple rule for derivatives. Sum rule: \(\frac{d}{dx}[f + g] = f' + g'\) — differentiate term by term.

Constant multiple: \(\frac{d}{dx}[c \cdot f] = c \cdot f'\) — constants factor out.

Example: \(\frac{d}{dx}[3x^2 - 4x + 1] = 6x - 4\).
Write the product rule and quotient rule. Product: \((fg)' = f'g + fg'\)

Quotient: \(\left(\frac{f}{g}\right)' = \frac{f'g - fg'}{g^2}\)

Mnemonic for quotient: "lo d-hi minus hi d-lo over lo squared."
Write the gradient descent update rule for a single parameter.
\[ w \leftarrow w - \alpha \cdot \frac{\partial L}{\partial w} \]
We subtract the gradient because gradients point uphill, and we want to go downhill (minimise loss). \(\alpha\) is the learning rate controlling step size.
What happens with a learning rate too high vs too low?
  • Too high \(\alpha\): overshoots the minimum, loss oscillates or diverges
  • Too low \(\alpha\): training is very slow — takes many steps to converge
  • Just right: smooth descent to the minimum
In practice: use a learning rate scheduler or adaptive optimisers (Adam, RMSprop) to tune α automatically.
What are the three main variants of gradient descent?
  • Batch GD: gradient over all training examples — exact but slow on large data
  • Stochastic GD (SGD): gradient from one random example — fast but noisy
  • Mini-batch GD: gradient over a small batch — best of both worlds, used in practice
PyTorch's torch.optim.SGD actually does mini-batch SGD by default.
When is gradient descent guaranteed to find the global minimum? Only for convex loss surfaces — functions where any line segment between two points lies above the curve. For convex losses (e.g. MSE for linear regression), GD converges to the unique global minimum.

For non-convex losses (neural networks), GD may get stuck in local minima or saddle points.
Given \(L(w)=w^2\), \(w_0=6\), \(\alpha=0.1\): compute one GD step. \(L'(w) = 2w\). At \(w_0 = 6\): \(L'(6) = 12\).
\[ w_1 = 6 - 0.1 \times 12 = 6 - 1.2 = 4.8 \]
After each step the parameter gets closer to the minimum at \(w^* = 0\).
What is a saddle point and why does it matter in deep learning? A saddle point is where \(\nabla L = 0\) but it is neither a minimum nor a maximum — the loss goes down in some directions and up in others (like a mountain pass).

In high-dimensional networks, saddle points are far more common than local minima. SGD's noise often helps escape them, which is one reason it works well in practice.
What advantage does Adam optimiser have over plain GD? Adam (Adaptive Moment Estimation) maintains a per-parameter adaptive learning rate based on first moment (mean gradient) and second moment (gradient variance).

Result: parameters that have sparse or small gradients get larger steps; parameters with large gradients get smaller steps. Much faster convergence in practice, with less tuning of α.
What is a partial derivative \(\partial f / \partial x_i\)? The derivative of \(f\) with respect to \(x_i\) while treating all other variables as constants.
\[ \frac{\partial f}{\partial x_i} = \lim_{h\to 0}\frac{f(\ldots,x_i+h,\ldots)-f(\ldots,x_i,\ldots)}{h} \]
It answers: "how much does \(f\) change if I nudge just \(x_i\)?"
Define the gradient \(\nabla f\) of a multivariable function. The gradient is a vector of all partial derivatives:
\[ \nabla f = \begin{bmatrix}\partial f/\partial x_1 \\ \partial f/\partial x_2 \\ \vdots \\ \partial f/\partial x_n\end{bmatrix} \]
It points in the direction of steepest ascent. Subtracting it moves us downhill.
Write the gradient descent update for a vector of weights \(\mathbf{w}\).
\[ \mathbf{w} \leftarrow \mathbf{w} - \alpha \nabla_{\mathbf{w}} L \]
Each weight \(w_i\) is updated by its own partial derivative \(\partial L/\partial w_i\). All weights update simultaneously using the gradient vector.
For \(f(x,y) = x^2 y + 3y\), compute \(\partial f/\partial x\) and \(\partial f/\partial y\). ∂f/∂x: treat \(y\) as constant.
\[ \frac{\partial f}{\partial x} = 2xy \]
∂f/∂y: treat \(x\) as constant.
\[ \frac{\partial f}{\partial y} = x^2 + 3 \]
What is the Jacobian matrix? For a vector-valued function \(\mathbf{f}: \mathbb{R}^n \to \mathbb{R}^m\), the Jacobian is the \(m\times n\) matrix of all first-order partial derivatives:
\[ J_{ij} = \frac{\partial f_i}{\partial x_j} \]
It generalises the gradient from scalar-to-vector to vector-to-vector functions. Used in backpropagation through vector layers.
State the chain rule for composite functions. If \(y = f(g(x))\), then:
\[ \frac{dy}{dx} = f'(g(x)) \cdot g'(x) \]
"Derivative of outer, evaluated at inner, times derivative of inner."

In Leibniz notation: \(\frac{dy}{dx} = \frac{dy}{du}\cdot\frac{du}{dx}\) where \(u = g(x)\).
Apply the chain rule to \(h(x) = \sin(x^2)\). Outer: \(\sin(u)\), derivative \(\cos(u)\).
Inner: \(u = x^2\), derivative \(2x\).

\[ h'(x) = \cos(x^2) \cdot 2x = 2x\cos(x^2) \]
How does the chain rule power backpropagation? A neural network is a composition of functions: \(L = \ell(f_n(\cdots f_2(f_1(\mathbf{x}))\cdots))\).

Backprop applies the chain rule from the output back to each layer:
\[ \frac{\partial L}{\partial w_1} = \frac{\partial L}{\partial a_n}\cdot\frac{\partial a_n}{\partial a_{n-1}}\cdots\frac{\partial a_2}{\partial a_1}\cdot\frac{\partial a_1}{\partial w_1} \]
Intermediate values cached in the forward pass are reused — no redundant computation.
Derive the derivative of the sigmoid function \(\sigma(x) = \frac{1}{1+e^{-x}}\). Using the quotient rule and chain rule:
\[ \sigma'(x) = \sigma(x)\big(1 - \sigma(x)\big) \]
Key values: \(\sigma(0) = 0.5\), so \(\sigma'(0) = 0.5 \times 0.5 = \mathbf{0.25}\).

The sigmoid's gradient vanishes for large |x| — this causes the vanishing gradient problem in deep networks.
What is the vanishing gradient problem and how is it addressed? In deep networks, backpropagating through many sigmoid layers multiplies many small gradients (<1). The gradient signal decays exponentially toward the early layers — early weights barely update.

Solutions:
  • Use ReLU — gradient is 1 for positive inputs, 0 otherwise — no saturation
  • Residual connections (ResNets) — provide gradient highways
  • Batch normalisation — stabilises activations
What is a computational graph in the context of autograd? A directed acyclic graph where:
  • Nodes = operations (add, multiply, sigmoid…)
  • Edges = data flow (tensors)
During the forward pass, the graph is built and intermediate values stored. During .backward(), the graph is traversed in reverse, applying the chain rule at each node.

PyTorch builds this graph dynamically — each forward pass can have a different graph structure.
Write the formula for Mean Squared Error (MSE) and explain its properties.
\[ \text{MSE} = \frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i)^2 \]
  • Differentiable everywhere — great for GD
  • Sensitive to outliers — large errors are squared, dominating the loss
  • Convex for linear models → guaranteed global minimum
Used for regression tasks.
Compare MAE and MSE. When would you choose each? MSE = \(\frac{1}{n}\sum(y_i-\hat{y}_i)^2\): squares errors → penalises outliers heavily. Smooth and differentiable everywhere.

MAE = \(\frac{1}{n}\sum|y_i-\hat{y}_i|\): linear in errors → robust to outliers. Not differentiable at 0 (use subgradient or Huber loss instead).

Choose MAE when outliers are expected and should not dominate training.
Write the binary cross-entropy loss and explain why we use it for classification.
\[ L = -\frac{1}{n}\sum_{i=1}^n\big[y_i\log\hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\big] \]
The log function heavily penalises confident wrong predictions: if the model predicts \(\hat{p}=0.001\) for a true class, \(-\log(0.001) \approx 6.9\) — a very large penalty.

For multi-class: categorical cross-entropy. Combined with softmax output, it has a clean gradient: \(\hat{p} - y\).
What properties should a loss function have for gradient-based training?
  • Differentiable (or sub-differentiable) — needed for GD updates
  • Non-negative — 0 only at the solution
  • Reflects the task objective — regression → MSE; classification → cross-entropy
  • Smooth enough for stable training
MSE as a classification loss is possible but poorly calibrated — cross-entropy is always preferred for probabilities.
What is the gradient of MSE with respect to weights \(\mathbf{w}\)? For \(L = \frac{1}{n}\|\mathbf{X}\mathbf{w}-\mathbf{y}\|^2\):
\[ \nabla_{\mathbf{w}}L = \frac{2}{n}\mathbf{X}^\top(\mathbf{X}\mathbf{w}-\mathbf{y}) \]
The term \((\mathbf{X}\mathbf{w}-\mathbf{y})\) is the residual — prediction error. We project it back through \(\mathbf{X}^\top\) to get the weight gradient.

In NumPy: (2/n) * X.T @ (X @ w - y).
Frame linear regression as an optimisation problem. Find weights \(\mathbf{w}\) and bias \(b\) that minimise the MSE:
\[ \min_{\mathbf{w},b} \;\frac{1}{n}\sum_{i=1}^n(y_i - \mathbf{w}^\top\mathbf{x}_i - b)^2 \]
This is a convex quadratic in \(\mathbf{w}\) — it has a unique global minimum reachable by GD or the Normal Equations.
State the Normal Equations for linear regression.
\[ \mathbf{w}^* = (\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y} \]
Derived by setting \(\nabla_{\mathbf{w}}L = 0\) and solving analytically. This is the exact, closed-form solution.

Limitation: inverting \(\mathbf{X}^\top\mathbf{X}\) is \(\mathcal{O}(p^3)\) — infeasible for millions of features. Use GD instead for large \(p\).
What is Ridge regression and what problem does it solve? Ridge adds an L2 regularisation term to the MSE:
\[ L_{\text{Ridge}} = \frac{1}{n}\|\mathbf{X}\mathbf{w}-\mathbf{y}\|^2 + \lambda\|\mathbf{w}\|^2 \]
  • Penalises large weights → shrinks them toward zero
  • Prevents overfitting when features are highly correlated
  • Makes \(\mathbf{X}^\top\mathbf{X}\) invertible even when singular
Closed form: \(\mathbf{w}^* = (\mathbf{X}^\top\mathbf{X}+\lambda\mathbf{I})^{-1}\mathbf{X}^\top\mathbf{y}\).
When would you prefer Gradient Descent over the Normal Equations?
  • Large number of features (p ≫ 1000): \((\mathbf{X}^\top\mathbf{X})^{-1}\) is \(\mathcal{O}(p^3)\) — prohibitive
  • Streaming / online learning: GD can update one batch at a time
  • Non-linear models (neural nets): no closed form exists — must use GD
  • Normal Equations best for small, dense datasets with moderate \(p\)
How does Week 2's projection formula connect to the Normal Equations? The Normal Equations \(\mathbf{w}^* = (\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y}\) give the projection of y onto the column space of X.

The prediction \(\hat{\mathbf{y}} = \mathbf{X}\mathbf{w}^* = \mathbf{X}(\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y}\) is exactly the orthogonal projection matrix \(P = \mathbf{X}(\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\) from linear algebra.

Linear regression = finding the projection of the target onto the feature column space.
Describe the computation inside a single neuron. Two steps:
  1. Linear combination: \(z = \mathbf{w}^\top\mathbf{x} + b\)
  2. Activation: \(a = \sigma(z)\)
\(z\) is the pre-activation (weighted sum + bias). \(a\) is the neuron's output after the non-linear squash.

Without the activation \(\sigma\), a neural network is just linear regression, no matter how many layers.
Compare Sigmoid, ReLU, and Tanh activation functions.
  • Sigmoid \(\sigma(x)=\frac{1}{1+e^{-x}}\): output (0,1), saturates for large |x| → vanishing gradients
  • ReLU \(\max(0,x)\): output [0,∞), no saturation for \(x>0\), sparse activations, most used in hidden layers
  • Tanh \(\frac{e^x-e^{-x}}{e^x+e^{-x}}\): output (-1,1), zero-centered, still saturates at extremes
ReLU is the default choice for hidden layers; Sigmoid for binary output.
Why can't we stack linear layers without activations? A composition of linear functions is still linear:
\[ W_2(W_1\mathbf{x}) = (W_2 W_1)\mathbf{x} = W_{\text{combined}}\mathbf{x} \]
No matter how many layers, the network collapses to a single matrix multiplication — it can only learn linear decision boundaries.

Non-linear activations break this collapse, allowing the network to approximate any continuous function (Universal Approximation Theorem).
Describe the forward pass through a 2-layer neural network. Given input \(\mathbf{x}\):
  1. \(\mathbf{z}^{(1)} = W^{(1)}\mathbf{x} + \mathbf{b}^{(1)}\)
  2. \(\mathbf{a}^{(1)} = \text{ReLU}(\mathbf{z}^{(1)})\)
  3. \(\mathbf{z}^{(2)} = W^{(2)}\mathbf{a}^{(1)} + \mathbf{b}^{(2)}\)
  4. \(\hat{y} = \sigma(\mathbf{z}^{(2)})\) (sigmoid for binary classification)
All intermediate values \(\mathbf{z}, \mathbf{a}\) are cached for backpropagation.
What is the derivative of ReLU? Why is it useful?
\[ \text{ReLU}'(x) = \begin{cases}1 & x > 0 \\ 0 & x \leq 0\end{cases} \]
The gradient is either 1 (no attenuation — no vanishing gradient for active units) or 0 (dead neuron). This binary gradient is much simpler and faster to compute than sigmoid's.

Dying ReLU problem: neurons stuck at 0 never recover. Fix: Leaky ReLU = \(\max(0.01x, x)\).
What is the softmax function and when is it used?
\[ \text{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_j e^{z_j}} \]
Converts a vector of raw scores (logits) into a probability distribution that sums to 1. Used in the output layer of multi-class classifiers.

Paired with categorical cross-entropy, the gradient simplifies to \(\hat{p} - y\) — elegant and numerically stable.
Implement gradient descent in NumPy for MSE loss. import numpy as np
w = np.zeros(p)
for _ in range(n_iters):
  pred = X @ w
  grad = (2/n) * X.T @ (pred - y)
  w -= lr * grad

Key: compute residual \(X\mathbf{w}-\mathbf{y}\), project through \(X^\top\) to get gradient, subtract scaled gradient.
How do you use scipy.optimize.minimize? from scipy.optimize import minimize
def loss(w): return np.mean((X @ w - y)**2)
res = minimize(loss, x0=np.zeros(p))
w_opt = res.x

Internally uses gradient-based methods (L-BFGS by default for smooth functions). Suitable for small–medium problems where you don't want to implement GD manually.
Write the basic PyTorch training loop structure. import torch
w = torch.tensor([0.0], requires_grad=True)
optim = torch.optim.Adam([w], lr=0.01)
for epoch in range(100):
  loss = ((X @ w - y)**2).mean()
  optim.zero_grad()
  loss.backward() # chain rule
  optim.step() # w -= lr * grad
Why must you call optim.zero_grad() before loss.backward()? PyTorch accumulates gradients in tensor.grad by default. Without zeroing, each .backward() adds to the previous gradient instead of replacing it.

The training loop must be: zero → forward → backward → step.

This accumulation behaviour is intentional for some use cases (e.g., gradient accumulation over mini-batches to simulate larger batches), but for standard training it must be cleared each step.
Implement the Normal Equations in NumPy. w_opt = np.linalg.inv(X.T @ X) @ X.T @ y

Or more numerically stable (avoids explicit inverse): w_opt = np.linalg.lstsq(X, y, rcond=None)[0]

lstsq uses SVD internally — handles rank-deficient cases gracefully and is always preferred over explicit inversion.
How do Weeks 1–3 combine to power modern ML?
  • Week 1 (Linear Algebra): data is vectors; models are matrices; predictions are \(W\mathbf{x}\)
  • Week 2 (Eigenvalues/PCA): understand data structure; compress without losing information; regression = projection
  • Week 3 (Calculus): measure loss; compute gradients; update weights via backprop
Together: represent data as vectors, build a model as matrices, optimise it with gradients. That is the mathematical core of every neural network.
How does gradient descent connect to least-squares projection? The Normal Equations \(\mathbf{w}^* = (\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y}\) and gradient descent both find the same answer — the orthogonal projection of \(\mathbf{y}\) onto the column space of \(\mathbf{X}\).

GD is an iterative journey toward that projection; the Normal Equations jump there in one shot. For large problems, the journey is cheaper than the jump.
How do PCA and gradient descent both appear inside a deep learning pipeline?
  • PCA / SVD: preprocessing step — reduce input dimensionality before feeding to the network; also used to initialise weight matrices (e.g., SVD of weight matrices in transformers)
  • Gradient descent: training step — backprop through every layer, update every weight
Also: the loss landscape's curvature is captured by the Hessian (second derivatives) — its eigenvectors/eigenvalues determine convergence speed of GD near the minimum.
How does the chain rule extend to matrices in neural networks? When layers process vectors/matrices, the chain rule becomes matrix multiplication of Jacobian matrices:
\[ \frac{\partial L}{\partial W^{(1)}} = \frac{\partial L}{\partial \mathbf{a}^{(2)}} \cdot \frac{\partial \mathbf{a}^{(2)}}{\partial \mathbf{z}^{(2)}} \cdot \frac{\partial \mathbf{z}^{(2)}}{\partial \mathbf{a}^{(1)}} \cdot \frac{\partial \mathbf{a}^{(1)}}{\partial W^{(1)}} \]
Autograd handles this automatically — you just define the forward computation and call .backward().
