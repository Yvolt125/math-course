What does \(A\vec{v}\) do geometrically? It moves \(\vec{v}\) to a new location — the matrix is a transformation machine. Applying \(A\) to every point simultaneously deforms the entire space according to one rule.
Name the 4 fundamental 2D geometric transformations.
  • Scaling — stretch/compress along axes: \(\begin{bmatrix}3&0\\0&2\end{bmatrix}\)
  • Rotation by θ: \(\begin{bmatrix}\cos\theta&-\sin\theta\\\sin\theta&\cos\theta\end{bmatrix}\)
  • Shear — slant space like a tilted deck of cards: \(\begin{bmatrix}1&1\\0&1\end{bmatrix}\)
  • Reflection — mirror across an axis: \(\begin{bmatrix}1&0\\0&-1\end{bmatrix}\)
What is the column space of a matrix? The set of all possible outputs — every vector \(A\vec{v}\) for every possible input \(\vec{v}\). It is where you can land. A full-rank \(m\times n\) matrix's column space fills all of \(\mathbb{R}^m\).
What is the null space of a matrix? All inputs squashed to zero: every \(\vec{v}\) where \(A\vec{v} = \vec{0}\). A non-trivial null space is the geometric signature of a singular matrix — those directions are permanently destroyed.
State the Rank-Nullity Theorem.
\[ \text{rank}(A) + \text{nullity}(A) = n \]
The number of independent output directions plus the number of input directions destroyed always equals the number of input dimensions n. You cannot gain information, only lose it.
How do neural network layers relate to geometric transformations? Each layer computes \(W\vec{x}\) (a matrix transformation) followed by a non-linear activation. The network is a chain of geometric deformations — rotating, stretching, bending input space — until data of different classes lands on opposite sides of a separating hyperplane.
Give the two definitions of the dot product. Algebraic:
\[ \vec{a}\cdot\vec{b} = a_1b_1 + a_2b_2 + \cdots + a_nb_n \]
Geometric:
\[ \vec{a}\cdot\vec{b} = \|\vec{a}\|\|\vec{b}\|\cos\theta \]
Algebraic = fast to compute; geometric = tells you what it means.
What does the sign of the dot product tell you?
  • > 0: θ < 90° — vectors point roughly the same way
  • = 0: θ = 90° — vectors are orthogonal (completely independent directions)
  • < 0: θ > 90° — vectors point in opposite directions
When are two vectors orthogonal? When their dot product equals zero: \(\vec{a}\cdot\vec{b} = 0\). Orthogonal vectors carry completely independent information — knowing one tells you nothing about the other. The mathematical ideal for ML features.
Write the formula for the norm (length) of a vector.
\[ \|\vec{v}\| = \sqrt{v_1^2 + v_2^2 + \cdots + v_n^2} \]
The norm measures vector length. For a unit vector: \(\|\vec{v}\| = 1\). In NumPy: np.linalg.norm(v).
What is cosine similarity and when do you use it?
\[ \cos(\theta) = \frac{\vec{a}\cdot\vec{b}}{\|\vec{a}\|\|\vec{b}\|} \]
A normalized alignment score between −1 and +1, independent of vector length. Used in NLP (word embeddings), recommender systems — anywhere "direction matters but magnitude doesn't."
Where does the dot product appear in ML?
  • Linear prediction: \(\hat{y} = \vec{w}\cdot\vec{x} + b\) — alignment of weights with input
  • Cosine similarity — document/word relevance
  • Attention (transformers): score(Q,K) = QKᵀ / √d_k — scaled dot product measures query-key alignment
Write the formula for the projection of \(\vec{a}\) onto \(\vec{b}\).
\[ \text{proj}_{\vec{b}}(\vec{a}) = \frac{\vec{a}\cdot\vec{b}}{\|\vec{b}\|^2}\,\vec{b} \]
The error \(\vec{a} - \text{proj}\) is always perpendicular to \(\vec{b}\). Perpendicularity is what "closest" means geometrically.
Why must the projection error be perpendicular to the subspace? The perpendicular foot is the closest point on the subspace to \(\vec{a}\). Any other direction of error would be farther away. "Closest" in Euclidean geometry always means "error is perpendicular."
State the Normal Equations. When \(A\vec{x} = \vec{b}\) has no exact solution, the best approximation \(\hat{x}\) satisfies:
\[ A^TA\,\hat{x} = A^T\vec{b} \]
Derived by requiring the error \(\vec{b} - A\hat{x}\) to be perpendicular to all columns of \(A\).
How does projection derive the linear regression formula? Real data has no exact solution, so we project \(\vec{y}\) onto the column space of \(X\). The closest fit satisfies the normal equations:
\[ \hat{w} = (X^TX)^{-1}X^T\vec{y} \]
The residual \(\vec{y} - X\hat{w}\) is perpendicular to every predictor — this is why OLS minimises the sum of squared errors.
What is an overdetermined system? A system \(A\vec{x}=\vec{b}\) with more equations than unknowns — more data points than parameters. No exact solution exists because \(\vec{b}\) is unlikely to lie in the column space of \(A\). Least squares projection finds the best approximation.
Project a = [4, 3] onto b = [1, 0].
Formula: \(\text{proj}_{\mathbf{b}}(\mathbf{a}) = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{b}\|^2} \mathbf{b}\) a·b = 4(1) + 3(0) = 4
‖b‖² = 1² + 0² = 1
proj = (4/1) × [1, 0] = [4, 0]
Projection onto the x-axis always gives just the x-component.
State the definition of an eigenvector and eigenvalue.
\[ A\vec{v} = \lambda\vec{v} \]
An eigenvector \(\vec{v}\) is only scaled by matrix \(A\) — never rotated. \(\lambda\) (lambda) is the eigenvalue: the scaling factor. The direction of \(\vec{v}\) is an invariant axis of the transformation.
Geometrically, what is special about an eigenvector? It is an invariant axis — the only directions the matrix does not rotate. Everything else gets swept to a new direction; eigenvectors only scale (or flip). They reveal the natural coordinate system of the transformation.
What does each eigenvalue value mean geometrically?
  • λ > 1: eigenvector is stretched
  • 0 < λ < 1: eigenvector is compressed
  • λ < 0: eigenvector is flipped (and scaled)
  • λ = 0: eigenvector is squashed to zero → matrix is singular
How do you find eigenvalues? State the characteristic equation. Rearrange \(A\vec{v} = \lambda\vec{v}\) to \((A-\lambda I)\vec{v}=\vec{0}\). For a non-zero solution, \((A-\lambda I)\) must be singular:
\[ \det(A - \lambda I) = 0 \]
Solving this polynomial gives all n eigenvalues of an n×n matrix.
Once you have eigenvalue \(\lambda\), how do you find its eigenvector? Substitute back and solve the homogeneous system:
\[ (A - \lambda I)\vec{v} = \vec{0} \]
Reduce to row echelon form; free variables give the eigenvector direction. Any scalar multiple of \(\vec{v}\) is also an eigenvector.
How do eigenvalues relate to trace and determinant?
\[ \text{tr}(A) = \lambda_1 + \lambda_2 + \cdots + \lambda_n \]
\[ \det(A) = \lambda_1 \cdot \lambda_2 \cdots \lambda_n \]
Therefore: \(\det(A) = 0 \Leftrightarrow\) at least one eigenvalue is zero — a zero eigenvalue is the algebraic fingerprint of singularity.
Are eigenvectors for distinct eigenvalues always independent? Yes. Eigenvectors for distinct eigenvalues are always linearly independent — they point in genuinely different invariant directions. Eigenvectors for the same eigenvalue can be combined freely (any linear combination is also an eigenvector for that λ).
Find the eigenvalues of:
\[ A = \begin{bmatrix} 3 & 1 \\ 0 & 2 \end{bmatrix} \]
Hint: solve det(A − λI) = 0. det\(\begin{bmatrix}3-λ&1\\0&2-λ\end{bmatrix}\) = (3−λ)(2−λ) − 0 = 0
(3−λ)(2−λ) = 0
λ₁ = 3, λ₂ = 2
Shortcut: upper/lower triangular → eigenvalues are the diagonal entries.
A 3×3 matrix has eigenvalues λ₁ = 1, λ₂ = 2, λ₃ = 4.
Find: (a) trace(A) and (b) det(A). (a) trace(A) = λ₁ + λ₂ + λ₃ = 1 + 2 + 4 = 7
(b) det(A) = λ₁ × λ₂ × λ₃ = 1 × 2 × 4 = 8
These identities hold for any square matrix.
What is a symmetric matrix? A matrix where \(A = A^T\) — identical to its own transpose. Every entry above the diagonal mirrors the one below.
Common in ML: any \(X^TX\), every covariance matrix, SVM kernel matrices — all symmetric.
State the Spectral Theorem for symmetric matrices. Every real symmetric matrix S guarantees:
  • All eigenvalues are real (never complex)
  • Eigenvectors for different eigenvalues are orthogonal
  • S is always fully diagonalizable
None of these are guaranteed for general matrices.
What is diagonalization? Write the decomposition.
\[ A = PDP^{-1} \]
D = diagonal matrix of eigenvalues; P = matrix of eigenvectors as columns. Geometrically: change to eigenvector basis → scale each axis independently → change back.
Write the eigendecomposition of a symmetric matrix. For symmetric S, eigenvectors are orthogonal so \(P^{-1} = P^T\):
\[ S = QDQ^T \]
Q = orthogonal matrix of unit-length eigenvectors; D = diagonal eigenvalues. "Rotate to eigenvector basis, scale each axis, rotate back."
How does diagonalization simplify computing \(A^n\)?
\[ A^n = PD^nP^{-1} \]
\(D^n\) just raises each diagonal entry to the nth power — trivial. Computing \(A^{1000}\) is as easy as computing \(A\) once you have the decomposition.
Distinguish positive definite (PD) from positive semi-definite (PSD).
  • PD: \(\vec{v}^TS\vec{v} > 0\) for all \(\vec{v}\neq 0\) → all λ > 0 → full rank, invertible, unique minimum
  • PSD: \(\vec{v}^TS\vec{v} \geq 0\) → all λ ≥ 0 → may be singular
Covariance matrices are always PSD. A PD loss surface guarantees a unique minimum.
What is an orthogonal matrix Q, and what equation defines it? A square matrix whose columns are unit-length and mutually orthogonal.
Key property: QᵀQ = QQᵀ = I
This means Q⁻¹ = Qᵀ — the inverse is free (just transpose).
Examples: rotation matrices, reflection matrices.
What geometric property does multiplying by an orthogonal matrix Q preserve? Q preserves lengths and angles.
For any vector v: ‖Qv‖ = ‖v‖ (norm unchanged).
For any pair: (Qu)·(Qv) = u·v (dot product unchanged).
Q performs a rigid-body transformation: rotation or reflection — no stretching or shearing.
Define the variance of a feature.
\[ \text{Var}(x) = \frac{1}{n}\sum_{i=1}^n (x_i - \bar{x})^2 \]
Average squared distance from the mean. High variance = spread out; low variance = clustered around the mean.
Define the covariance between two features.
\[ \text{Cov}(x,y) = \frac{1}{n}\sum_{i=1}^n (x_i-\bar{x})(y_i-\bar{y}) \]
  • > 0: features rise and fall together (correlated)
  • = 0: features are uncorrelated
  • < 0: features move in opposite directions (anti-correlated)
Write the formula for the covariance matrix Σ. For centered data \(\tilde{X}\) (mean subtracted):
\[ \Sigma = \frac{1}{n}\tilde{X}^T\tilde{X} \]
A p×p symmetric PSD matrix. Diagonal = variances; off-diagonal = covariances between feature pairs.
How do you read the covariance matrix Σ?
  • Diagonal entries: variance of each feature (σᵢ²)
  • Off-diagonal entries: covariance between pairs of features
  • All off-diagonal = 0: features are uncorrelated — clean, independent directions
  • Large off-diagonal values: highly correlated features → near-singular Σ
What does the covariance matrix look like geometrically? It defines an ellipsoid in feature space — a stretched, rotated egg showing where the data lives.
  • Eigenvectors of Σ = principal axes of the ellipsoid
  • Eigenvalues = variance (spread) along each axis
PCA finds and uses these axes.
What is multicollinearity and why is it dangerous? Highly correlated features make \(X^TX\) (proportional to Σ) nearly singular. Inverting a near-singular matrix amplifies floating-point errors explosively — small noise causes wildly unstable weight estimates.
Fix: Ridge regression adds λI to X^TX before inverting, ensuring positive definiteness.
What problem does PCA solve? Many features are correlated — overlapping information. PCA finds the directions of maximum variance in the data and projects onto a lower-dimensional space along those directions. Goal: retain maximum information in fewer dimensions.
List the 6 steps of the PCA algorithm.
  1. Center the data: \(\tilde{X} = X - \bar{X}\)
  2. Compute covariance matrix: \(\Sigma = \frac{1}{n}\tilde{X}^T\tilde{X}\)
  3. Eigendecompose Σ: find eigenvectors and eigenvalues
  4. Sort by eigenvalue descending
  5. Choose k components to keep
  6. Project: \(Z = \tilde{X}Q_k\)
What is a principal component? An eigenvector of the covariance matrix Σ, sorted by eigenvalue.
  • PC1: direction of maximum variance in the data
  • PC2: direction of max remaining variance, orthogonal to PC1
  • Each PC is orthogonal to all others (Spectral Theorem guarantees this for symmetric Σ)
What is the explained variance ratio for PCᵢ?
\[ \frac{\lambda_i}{\lambda_1 + \lambda_2 + \cdots + \lambda_p} \]
Fraction of total variance captured by component i. Sum the top k ratios for total variance retained. Common rule: keep enough components to explain ≥ 95% of total variance.
Why must you center the data before PCA? Without centering, the first PC would point toward the data's centroid (mean) rather than the direction of maximum spread. Centering removes the mean so PCA finds variance directions, not location directions.
Name 4 practical uses of PCA.
  • Dimensionality reduction: reduce 500 features → 20 PCs before a classifier
  • Visualization: project to top 2–3 PCs for human inspection
  • Noise reduction: small eigenvalues ≈ noise → discard them
  • Remove multicollinearity: project correlated features onto uncorrelated PCs before regression
Why is PCA computed via SVD in practice, not the covariance matrix? SVD of centered data \(\tilde{X} = U\Sigma V^T\) gives principal component directions as columns of V directly — without forming \(\tilde{X}^T\tilde{X}\). More numerically stable — forming \(X^TX\) explicitly can amplify rounding errors.
sklearn.decomposition.PCA uses SVD internally.
A dataset has 4 features. After PCA, the covariance matrix eigenvalues are [6, 3, 1, 0]. What percentage of variance does PC1 explain? Total variance = 6 + 3 + 1 + 0 = 10
PC1 explained variance ratio = 6/10 = 60%
PC1 + PC2 = (6+3)/10 = 90%
PC1+PC2+PC3 = 10/10 = 100% (λ₄=0 adds nothing)
Compute a dot product, matrix multiply, and vector norm in NumPy. np.dot(a, b) or a @ b — dot product / matrix multiply
np.linalg.norm(v) — vector length ‖v‖
cos_sim = np.dot(a,b) / (np.linalg.norm(a) * np.linalg.norm(b))
When should you use np.linalg.eig vs np.linalg.eigh?
  • np.linalg.eig(A) — any square matrix; may return complex numbers; unsorted
  • np.linalg.eigh(S) — symmetric matrices only; always real; returns sorted ascending; more numerically stable
Always use eigh for covariance matrices.
Code the projection of vector a onto vector b in NumPy. proj = (np.dot(a, b) / np.dot(b, b)) * b

Example: a = [3,4], b = [1,0] → proj = [3,0]
Write the core PCA from scratch steps in NumPy. X_c = X - X.mean(axis=0)         # center
cov = X_c.T @ X_c / n             # covariance matrix
vals, vecs = np.linalg.eigh(cov)   # eigendecompose
order = np.argsort(vals)[::-1]     # sort descending
Q_k = vecs[:, order[:k]]           # top k eigenvectors
Z = X_c @ Q_k                      # project to k dims
Run PCA using scikit-learn. from sklearn.decomposition import PCA
pca = PCA(n_components=2)
Z = pca.fit_transform(X)
pca.explained_variance_ratio_  # fraction per PC
pca.components_                # PC vectors (rows)

Auto-select: PCA(n_components=0.95) keeps minimum PCs for 95% variance.
Compute the covariance matrix in NumPy. np.cov(X.T) — divides by n−1 (Bessel's correction) by default.

For exact \(\frac{1}{n}\tilde{X}^T\tilde{X}\):
X_c = X - X.mean(axis=0)
cov = X_c.T @ X_c / n

Why are the eigenvectors of Σ the principal components? Eigenvectors of the covariance matrix are the directions along which data has maximum variance — each eigenvalue equals the variance in that direction. The largest-eigenvalue eigenvector = most variance = PC1. Projecting onto these directions retains maximum information in fewer dimensions.
A covariance matrix Σ has a zero eigenvalue — what does this mean? Some feature (or combination of features) has zero variance — perfectly predictable from others. The matrix is singular: det(Σ)=0, no inverse, linear regression fails. This is the algebraic signature of perfect multicollinearity — connecting Week 1 (singularity) to Week 2 (covariance/PCA).
Why is \(X^TX\) always symmetric and positive semi-definite? Symmetric: \((X^TX)^T = X^TX\) ✓
PSD: for any vector \(\vec{v}\):
\[ \vec{v}^T(X^TX)\vec{v} = \|X\vec{v}\|^2 \geq 0 \]
A squared norm is always non-negative. This guarantees all eigenvalues of \(X^TX\) are ≥ 0.
Distinguish orthogonal, uncorrelated, and statistically independent.
  • Orthogonal vectors: dot product = 0 (geometric property)
  • Uncorrelated features: covariance = 0 (statistical property)
  • Statistically independent: knowing one gives zero information about the other (stronger than uncorrelated)
For Gaussian data, uncorrelated = independent. In general, uncorrelated does not imply independent.
