Chapter 1
Series & DataFrames

Pandas gives you two core data structures: a Series (a labelled 1-D array) and a DataFrame (a labelled 2-D table). Nearly every data task you do lives inside one of these two objects.

The Series

Think of a Series as a single column from a spreadsheet — values with an index label on each row.

When you'll encounter a Series in practice You rarely create a Series manually. They appear as the natural output of common operations:
  • df['sales'] — selecting one column returns a Series
  • df.groupby('region')['sales'].sum() — a groupby aggregate is a Series
  • df['region'].value_counts() — frequency count is a Series
  • df.isnull().sum() — missing count per column is a Series
You apply Series methods (.mean(), .str.upper(), .fillna()) to individual columns. When you need multiple columns together, that's when you work with a DataFrame.

Mental model Think of a DataFrame as a Python dictionary where each key is a column name and each value is a list of equal length. Pandas wraps that dictionary with powerful query, transform, and output tools.

Accessing Columns

Chapter 2
Loading Data

In practice you almost never build a DataFrame by hand — you load it from a file. Pandas can read CSV, Excel, JSON, SQL, HTML, Parquet, and more.

Reading CSV

# Basic load
df = pd.read_csv('sales.csv')

# Common options
df = pd.read_csv(
    'sales.csv',
    sep=';',             # delimiter (default comma)
    header=0,            # row index of column names (default 0)
    index_col='id',      # use this column as the row index
    usecols=['name','sales','month'],  # only load these columns
    dtype={'sales': float},           # force data type
    parse_dates=['date'],             # auto-parse date column
    nrows=1000,          # only read first 1000 rows (great for big files)
    skiprows=[1, 2],     # skip specific rows
    na_values=['N/A', '--', 'missing']  # treat these as NaN
)

Reading from SQL

Saving Data

df.to_csv('output.csv', index=False)          # index=False avoids extra column
df.to_excel('output.xlsx', index=False, sheet_name='Results')
df.to_json('output.json', orient='records', indent=2)
Real-world tip — encoding errors Files from Windows often use cp1252 encoding. If you get a UnicodeDecodeError, add encoding='cp1252' or encoding='latin-1' to your read call.

Combining Multiple Files

A very common task: load every CSV in a folder into one DataFrame.

from pathlib import Path

folder = Path('monthly_reports')
frames = []
for f in folder.glob('*.csv'):
    tmp = pd.read_csv(f)
    tmp['source_file'] = f.name   # track which file each row came from
    frames.append(tmp)

df = pd.concat(frames, ignore_index=True)
print(f"Loaded {len(frames)} files, {len(df)} rows total")

Inspecting Data

Before any analysis, you need to understand what you have. These are the first commands you run on any new dataset.

df.head(10)    # first 10 rows (default 5)

df.shape       # (rows, columns) tuple
df.columns     # column names
df.dtypes      # data type of each column

Statistical Summary

df.describe()      # count, mean, std, min, quartiles, max for numeric cols
df.describe(include='object')   # frequency summary for text columns
df['sales'].value_counts()      # frequency of each unique value
df['region'].nunique()          # number of unique values (4)
df['region'].unique()           # array of unique values
df.info() output — what to look for
  • Columns where non-null count is less than total rows → missing data
  • A numeric column stored as object → likely has stray text; needs cleaning
  • Dates stored as object → use pd.to_datetime() to convert

Checking for Missing Data

df.isnull().sum()          # count of NaN per column
df.isnull().sum() / len(df) * 100  # % missing per column

# Heatmap-style view
df.isnull().any(axis=1)    # True for rows with ANY missing value
df[df.isnull().any(axis=1)]  # show those rows

Chapter 4
Filtering & Selecting

This is where most of your day-to-day Pandas work lives. There are three ways to select data: boolean masking, .loc, and .iloc.

Boolean Masking

Create a True/False mask with a condition, then pass it to the DataFrame:

mask = df['sales'] > 13000
print(mask)
# 0    False
# 1     True
# 2    False
# 3     True
# 4     True

df[mask]   # only rows where mask is True

# One-liner (more common in practice)
df[df['sales'] > 13000]

# Multiple conditions — use & (and) | (or), always wrap in ()
df[(df['region'] == 'South') & (df['sales'] > 12000)]
df[(df['month'] == 'Jan') | (df['month'] == 'Feb')]

# Negate
df[~(df['region'] == 'North')]    # same as != 'North'

# isin — check membership
df[df['region'].isin(['North', 'East'])]

# String filtering
df[df['name'].str.startswith('A')]
df[df['name'].str.contains('a', case=False)]

loc and iloc

loc vs iloc — choosing in practice
Use .loc when…Use .iloc when…
You know the column name: df.loc[:, 'sales']You want a position: "give me the 3rd row" → df.iloc[2]
Filtering with a condition: df.loc[df['region']=='South', 'sales']Building a generic function where you don't know column names
Selecting a named slice of rows and columns in one goSlicing the last N rows: df.iloc[-10:]
In day-to-day analysis you'll use .loc far more. .iloc is mainly useful for "first row / last row" or writing reusable utilities that work on any DataFrame.

# .loc — label-based
df.loc[1]               # row with label 1 (a Series)
df.loc[1, 'sales']      # single cell → 15800
df.loc[1:3, 'name':'sales']  # slice of rows AND cols (inclusive both ends!)
df.loc[df['region']=='South', 'sales']  # boolean mask + column name

# .iloc — position-based (like Python list indexing)
df.iloc[0]              # first row
df.iloc[-1]             # last row
df.iloc[0, 2]           # row 0, column 2 → 12400
df.iloc[1:4, 1:3]       # rows 1,2,3 — cols 1,2 (exclusive end!)
df.iloc[:, 2]           # all rows, column index 2
loc vs iloc slicing gotcha .loc[1:3] includes row 3. .iloc[1:3] excludes position 3. This follows the standard Python convention for iloc but not for loc.

The query() Method

.query() accepts a string expression — more readable for complex filters:

df.query("sales > 13000 and region == 'South'")
df.query("month in ['Jan', 'Feb'] and sales > 10000")

# Use @ to reference Python variables
threshold = 14000
df.query("sales > @threshold")
Chapter 5
Modifying Data

Creating new columns, transforming existing ones, handling dates, and applying custom functions — this is where you build the features your analysis needs.

Creating & Modifying Columns

# New column from arithmetic
df['commission'] = df['sales'] * 0.08

# New column from string operations
df['name_upper'] = df['name'].str.upper()

# New column from conditional (np.where is fast)
import numpy as np
df['tier'] = np.where(df['sales'] >= 15000, 'High', 'Standard')

# New column from multiple conditions — use np.select
conditions = [
    df['sales'] >= 18000,
    df['sales'] >= 13000,
]
choices = ['Platinum', 'Gold']
df['tier'] = np.select(conditions, choices, default='Silver')

Applying Functions

# .apply() on a column (element-wise)
df['label'] = df['sales'].apply(lambda x: f"${x:,}")

# .apply() on the whole DataFrame (row-wise, slower)
def classify(row):
    if row['region'] == 'South' and row['sales'] > 14000:
        return 'Target'
    return 'Other'

df['flag'] = df.apply(classify, axis=1)

# .map() — replace values using a dict (great for encoding)
region_code = {'North': 1, 'South': 2, 'East': 3, 'West': 4}
df['region_id'] = df['region'].map(region_code)

Working with Dates

Date handling is one of Pandas' biggest strengths.

df['date'] = pd.to_datetime(df['date'])       # parse strings to datetime
df['date'] = pd.to_datetime(df['date'], format='%d/%m/%Y')  # specify format

# Extract components
df['year']    = df['date'].dt.year
df['month']   = df['date'].dt.month
df['weekday'] = df['date'].dt.day_name()   # 'Monday', 'Tuesday', ...

# Date arithmetic
df['days_since'] = (pd.Timestamp.today() - df['date']).dt.days

# Filter by date
df[df['date'] >= '2024-01-01']
df[df['date'].between('2024-01-01', '2024-03-31')]

# Resample — aggregate by time period (needs datetime index)
df = df.set_index('date')
monthly = df['sales'].resample('ME').sum()   # month-end totals

Renaming & Dropping

df.rename(columns={'sales': 'revenue', 'name': 'rep_name'}, inplace=True)

# Drop columns
df.drop(columns=['commission', 'label'], inplace=True)

# Drop rows by index
df.drop(index=[0, 2], inplace=True)

# Drop rows based on condition
df = df[df['sales'] > 0]   # cleaner than drop in most cases

# Reset index after drops
df = df.reset_index(drop=True)
inplace=True vs reassignment Both do the same thing. df.drop(..., inplace=True) modifies df directly. df = df.drop(...) creates a new object and rebinds the name. Most experienced analysts prefer the reassignment style as it is safer and easier to reason about in complex pipelines.

Type Conversion

df['sales'] = df['sales'].astype(float)
df['date']  = pd.to_datetime(df['date'])
df['region'] = df['region'].astype('category')  # memory efficient for low-cardinality

# Handle columns that have mixed types (e.g. "15,000" strings)
df['sales'] = df['sales'].str.replace(',', '').astype(float)

grouped = df.groupby('region')

grouped['sales'].sum()        # total sales per region
grouped['sales'].mean()       # average
grouped['sales'].count()      # count rows per group
grouped['sales'].agg(['sum', 'mean', 'max', 'count'])

# Multiple columns
df.groupby('region')[['sales', 'commission']].sum()

# Multiple group keys
df.groupby(['region', 'month'])['sales'].sum()

# Named aggregations (Pandas ≥ 0.25)
result = df.groupby('region').agg(
    total_sales   = ('sales', 'sum'),
    avg_sales     = ('sales', 'mean'),
    max_sales     = ('sales', 'max'),
    rep_count     = ('name', 'count')
)
print(result)

Transform — Keep the Original Shape

.transform() computes group-level values but keeps each row's original index. Perfect for adding a group-level column back to the original DataFrame.

# Add a column: each rep's share of their region's total sales
df['region_total']   = df.groupby('region')['sales'].transform('sum')
df['region_share_%'] = df['sales'] / df['region_total'] * 100

# Add regional rank (1 = top performer in region)
df['rank_in_region'] = df.groupby('region')['sales'].rank(ascending=False)
groupby vs pivot_table — when to use each
Use groupby when…Use pivot_table when…
You want a list of aggregated values: total sales per regionYou want a 2-D grid: sales per region × month side by side
You need multiple aggregations in one step with agg()You want to share the result as a formatted report table
You'll do further computation on the resultYou want row/column totals with margins=True
Real example: "What are total sales per region?" → groupby. "Show me each region's sales for each month in a grid" → pivot_table.

Pivot Tables

Pivot tables reorganise grouped data into a 2-D layout — rows are one dimension, columns are another.

pivot = df.pivot_table(
    values='sales',
    index='region',
    columns='month',
    aggfunc='sum',
    fill_value=0,      # replace NaN with 0
    margins=True       # add row/column totals
)
print(pivot)
groupby + unstack = pivot df.groupby(['region','month'])['sales'].sum().unstack('month') produces the same result as a pivot table. Use whichever reads more clearly.

Merging & Joining

Real datasets are rarely in a single table. pd.merge() lets you combine DataFrames using a shared key column — just like SQL JOIN.

merge vs concat — how to choose
Use merge when…Use concat when…
Two tables share a key: orders + customers linked by customer_idSame columns, different rows: Jan + Feb + Mar CSVs stacked into one table
You're enriching a table with a lookup (add rep names, region codes, product details)You're combining results from different time periods or data sources
SQL JOIN equivalent — rows are aligned by a shared valueSQL UNION ALL equivalent — rows are just appended
Quick test: do the tables have the same rows described from different angles (linked by ID)? → merge. Do they have the same columns but different rows? → concat.

pd.merge(df_sales, df_reps, on='rep_id', how='inner')   # default: only matching rows

pd.merge(df_sales, df_targets, left_on='rep_id', right_on='id')

# Merge on multiple keys
pd.merge(df_a, df_b, on=['rep_id', 'month'])

# After merge, check for unexpected nulls
merged.isnull().sum()   # NaNs in right-only columns → left join left some unmatched

Concatenating DataFrames

pd.concat() stacks DataFrames vertically (more rows) or horizontally (more columns).

# Stack rows (same columns)
all_months = pd.concat([df_jan, df_feb, df_mar], ignore_index=True)

# Side by side (same number of rows)
df_combined = pd.concat([df_info, df_scores], axis=1)

# concat handles mismatched columns — fills missing with NaN
# merge is for aligning on a key value
# Don't use concat when you mean merge
Diagnosing bad merges
  • Row count exploded after merge → duplicate keys in one or both tables. Use df.duplicated(subset='key').sum() to check.
  • Row count shrank unexpectedly → key values don't match (different types, extra whitespace). Use df['key'].dtype and df['key'].str.strip().

Data often arrives in the wrong shape for analysis. The two most common transformations are melt (wide → long) and pivot (long → wide).

Wide vs Long Format

Wide format

Easy to read, hard to filter/group

Long format

repmonthsales
0AliceJan12400
1AliceFeb9800
2AliceMar14200

Preferred for groupby and plotting

# Wide → Long: melt
df_long = df_wide.melt(
    id_vars=['rep'],          # columns to keep as-is
    value_vars=['Jan','Feb','Mar'],  # columns to melt into rows
    var_name='month',         # new column name for the variable
    value_name='sales'        # new column name for the values
)

# Long → Wide: pivot
df_wide = df_long.pivot(
    index='rep',
    columns='month',
    values='sales'
)
df_wide.columns.name = None  # remove 'month' label from column axis

Stack & Unstack

# Rotate innermost column level → row level (pivot to long)
df.stack()

# Rotate innermost row level → column level (long to wide)
df.unstack()

# Practical use: flatten multi-level index after groupby
result = df.groupby(['region','month'])['sales'].sum()
# result.index is a MultiIndex (region, month)
result_wide = result.unstack('month')  # month levels become columns

Sorting

df.sort_values('sales', ascending=False)
df.sort_values(['region', 'sales'], ascending=[True, False])  # multi-key
df.sort_index()  # sort by row index
Chapter 9
Real-World Workflows

Here are four complete end-to-end patterns covering tasks analysts do every day.

Workflow 1 — Sales Report from Multiple Files

from pathlib import Path
import pandas as pd

# 1. Load all monthly CSVs
frames = []
for f in Path('data/monthly').glob('*.csv'):
    tmp = pd.read_csv(f, parse_dates=['date'])
    tmp['month'] = f.stem          # filename is the month name
    frames.append(tmp)
df = pd.concat(frames, ignore_index=True)

# 2. Clean
df.columns = df.columns.str.strip().str.lower().str.replace(' ', '_')
df['sales'] = pd.to_numeric(df['sales'], errors='coerce')
df = df.dropna(subset=['sales', 'rep_id'])

# 3. Enrich — add rep details from lookup table
reps = pd.read_csv('data/reps.csv')
df = pd.merge(df, reps[['rep_id','name','region']], on='rep_id', how='left')

# 4. Aggregate
summary = df.groupby(['region', 'month']).agg(
    total_sales = ('sales', 'sum'),
    rep_count   = ('rep_id', 'nunique'),
    avg_per_rep = ('sales', 'mean')
).round(2).reset_index()

# 5. Save
summary.to_csv('outputs/regional_summary.csv', index=False)
print("Done.", summary.shape)

Workflow 2 — Customer Churn Flag

import pandas as pd

orders = pd.read_csv('orders.csv', parse_dates=['order_date'])

# Last purchase date per customer
last_order = orders.groupby('customer_id')['order_date'].max().reset_index()
last_order.columns = ['customer_id', 'last_order_date']

# Days since last purchase
today = pd.Timestamp.today().normalize()
last_order['days_inactive'] = (today - last_order['last_order_date']).dt.days

# Flag customers inactive for >90 days
last_order['at_risk'] = last_order['days_inactive'] > 90

at_risk = last_order[last_order['at_risk']].sort_values('days_inactive', ascending=False)
print(f"{len(at_risk)} customers at risk of churn")
at_risk.to_csv('at_risk_customers.csv', index=False)

Workflow 4 — Deduplication & Canonical Merge

df = pd.read_csv('contacts.csv')

# 1. Standardise before dedup
df['email'] = df['email'].str.lower().str.strip()
df['phone'] = df['phone'].str.replace(r'\D', '', regex=True)  # digits only
