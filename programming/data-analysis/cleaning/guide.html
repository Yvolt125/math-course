Chapter 1
Auditing a Dataset

The first step in any data project is a data audit — a systematic inspection to understand what is wrong before you fix it. Cleaning without auditing leads to silent errors.

import pandas as pd
import numpy as np

df = pd.read_csv('raw_sales.csv')

# ── Shape & types ───────────────────────────────────────────────────
print(df.shape)
print(df.dtypes)
df.info()

# ── Missing data ────────────────────────────────────────────────────
missing = df.isnull().sum()
missing_pct = missing / len(df) * 100
print(pd.DataFrame({'missing': missing, 'pct': missing_pct.round(1)}))

# ── Duplicate rows ───────────────────────────────────────────────────
print(f"Duplicates: {df.duplicated().sum()}")
print(df[df.duplicated(keep=False)].head())   # show all copies

# ── Value ranges ─────────────────────────────────────────────────────
print(df.describe())

# ── Unexpected values in categoricals ────────────────────────────────
for col in df.select_dtypes('object').columns:
    print(f"\n{col}: {df[col].nunique()} unique")
    print(df[col].value_counts().head(10))
Common issues to look for
  • Columns that should be numeric stored as object (contains non-numeric text)
  • Date columns stored as strings
  • Categories with typos: "North", "north", "NORTH" treated as different values
  • Negative values where they make no sense (e.g. negative age, negative quantity)
  • Future dates in columns that should be historical
  • Zip codes / IDs stored as floats (losing leading zeros)
Chapter 2
Missing Values
Drop rows vs impute — when to choose which
SituationStrategyReal example
Less than 1–2% of rows missing, random patternDrop rowsA few customers skipped the age field at checkout
Important numeric column, missing randomlyImpute with medianHousehold income survey — median is robust to outliers
Missing because of another column (e.g. no purchase = no amount)Impute with constant (0 or "None")Transaction amount is NaN because customer never bought
Missing pattern contains signalKeep + add was_missing flagCredit applications — missing income field predicts default risk
High missingness (>50% of column)Drop columnOptional survey field almost nobody answered

Three strategies — choose carefully

df.isnull().sum()                  # count per column
df.isnull().mean() * 100           # percentage

# ── Strategy 1: Drop ─────────────────────────────────────────────────
# Drop rows where a specific column is null (key column)
df = df.dropna(subset=['sale_id', 'date'])

# Drop columns with too many missing values (>40%)
threshold = 0.4
cols_to_drop = df.columns[df.isnull().mean() > threshold]
df = df.drop(columns=cols_to_drop)

# ── Strategy 2: Fill with a fixed value ──────────────────────────────
df['region'].fillna('Unknown', inplace=True)
df['discount'].fillna(0, inplace=True)

# ── Strategy 3: Statistical imputation ───────────────────────────────
df['sales'].fillna(df['sales'].median(), inplace=True)
df['age'].fillna(df['age'].mean(), inplace=True)

# Group-level imputation (fill missing sales with regional mean)
df['sales'] = df.groupby('region')['sales'].transform(
    lambda x: x.fillna(x.mean())
)

# ── Forward fill (time-series: carry last value forward) ─────────────
df = df.sort_values('date')
df['price'] = df['price'].ffill()
df['price'] = df['price'].bfill()  # backward fill
Imputation is an assumption Filling with the mean assumes the data is missing randomly and that the mean is a reasonable substitute. That assumption can be wrong. Always document what you imputed and why, and consider creating a binary was_missing flag column so models can learn from the missingness pattern.

Advanced imputation with scikit-learn

from sklearn.impute import SimpleImputer, KNNImputer

# Simple imputer
imp = SimpleImputer(strategy='median')   # or 'mean', 'most_frequent', 'constant'
df[numeric_cols] = imp.fit_transform(df[numeric_cols])

# KNN imputer — fills based on k nearest neighbours
knn_imp = KNNImputer(n_neighbors=5)
df[numeric_cols] = knn_imp.fit_transform(df[numeric_cols])

# Fuzzy matching for larger datasets (requires rapidfuzz package)
from rapidfuzz import process, fuzz

valid = ['North', 'South', 'East', 'West']
def correct_region(val):
    match, score, _ = process.extractOne(val, valid, scorer=fuzz.WRatio)
    return match if score > 80 else val

df['region'] = df['region'].apply(correct_region)

Chapter 4
Outlier Detection

Outliers can skew your analysis, break models, or be the most interesting data points — context determines which. First detect them, then decide what to do.

Which outlier detection method should I use?
MethodUse whenReal example
IQR (1.5 × IQR rule)Data is not normally distributed; quick sanity check on numeric columnsSales figures with a few huge enterprise deals skewing the distribution
Z-score (±3 std)Data is roughly normal; outlier = statistically extreme valueWebsite load times — flag requests taking >3 std above mean
Isolation ForestMulti-dimensional data; you want ML to find anomalies automaticallyFraud detection — unusual combination of transaction amount, location, time
Domain ruleYou know the valid range from business logicAge must be 0–120; negative revenue is always an error

IQR method

Q1  = df['sales'].quantile(0.25)
Q3  = df['sales'].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

outliers = df[(df['sales'] < lower) | (df['sales'] > upper)]
print(f"Outliers: {len(outliers)} rows")
print(outliers[['rep_name','sales']])

# Remove outliers
df_clean = df[(df['sales'] >= lower) & (df['sales'] <= upper)]

# Or cap/clip (Winsorisation)
df['sales_capped'] = df['sales'].clip(lower=lower, upper=upper)

Z-score method

from scipy import stats
z_scores = stats.zscore(df['sales'].dropna())
outliers_z = df[np.abs(z_scores) > 3]   # beyond 3 std from mean

Isolation Forest (multivariate)

from sklearn.ensemble import IsolationForest

features = df[['sales', 'hours_worked', 'deals_closed']].dropna()
iso = IsolationForest(contamination=0.05, random_state=42)  # expect 5% outliers
labels = iso.fit_predict(features)   # -1 = outlier, 1 = normal
df.loc[features.index, 'is_outlier'] = labels == -1

print(df['is_outlier'].value_counts())
What to do with outliers
  • Investigate first — is it a data entry error, a genuine exceptional event, or a fraud signal?
  • Remove if they are clearly erroneous (e.g. age = 999)
  • Cap/Winsorise if you want to keep the row but limit influence
  • Flag and keep if they are genuine but you want models to treat them differently
  • Use robust statistics (median, IQR) if outliers are expected

Chapter 6
Encoding Categorical Variables

Machine learning models only understand numbers. Categorical columns must be encoded.

Which encoding method should I use?
MethodUse whenReal example
Label Encoding (integers)Ordinal data — the category has a natural orderEducation level: High School=0, Undergrad=1, Masters=2, PhD=3
One-Hot EncodingNominal data, low cardinality (<15 categories), linear modelsCountry of purchase: UK, US, DE → three binary columns
Target Encoding (mean encode)High cardinality nominal data, tree-based modelsCity (1000+ values) → replace each city with its mean conversion rate
Binary / Frequency EncodingHigh cardinality, memory-constrained, or you want a middle groundProduct SKU with 10k values — one-hot would explode column count

Label Encoding

Replace each category with an integer. Only use for ordinal data (where order matters).

from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()
df['tier_encoded'] = le.fit_transform(df['tier'])   # Silver→0, Gold→1, Platinum→2

# Manual mapping (explicit order)
tier_map = {'Silver': 0, 'Gold': 1, 'Platinum': 2}
df['tier_enc'] = df['tier'].map(tier_map)

One-Hot Encoding

Create a binary column for each category. Use for nominal data (no order). Avoid for high-cardinality columns.

# pandas get_dummies
region_dummies = pd.get_dummies(df['region'], prefix='region', drop_first=True)
df = pd.concat([df, region_dummies], axis=1)
df.drop(columns=['region'], inplace=True)

# sklearn OneHotEncoder (for pipelines)
from sklearn.preprocessing import OneHotEncoder
enc = OneHotEncoder(drop='first', sparse_output=False)
encoded = enc.fit_transform(df[['region']])
enc_df = pd.DataFrame(encoded, columns=enc.get_feature_names_out())
df = pd.concat([df.reset_index(drop=True), enc_df], axis=1)

Target Encoding

Replace each category with the mean of the target variable for that category. Useful for high-cardinality features. Be careful of data leakage.

means = df.groupby('region')['sales'].mean()
df['region_encoded'] = df['region'].map(means)
Chapter 7
Feature Engineering

Feature engineering creates new columns that make patterns more visible to models — or to your own analysis.

Scaling — putting numbers on the same scale

from sklearn.preprocessing import StandardScaler, MinMaxScaler

numeric = ['sales', 'hours_worked', 'deals_closed']

# StandardScaler: mean=0, std=1 (for algorithms sensitive to scale)
scaler = StandardScaler()
df[numeric] = scaler.fit_transform(df[numeric])

# MinMaxScaler: range [0, 1]
mm = MinMaxScaler()
df[numeric] = mm.fit_transform(df[numeric])

# Manual z-score (no sklearn)
df['sales_z'] = (df['sales'] - df['sales'].mean()) / df['sales'].std()

Binning — convert continuous to categories

df['sales_band'] = pd.cut(
    df['sales'],
    bins=[0, 10000, 15000, 25000, float('inf')],
    labels=['Low', 'Medium', 'High', 'Top']
)

# Equal-frequency bins (each bin has the same number of rows)
df['sales_quartile'] = pd.qcut(df['sales'], q=4, labels=['Q1','Q2','Q3','Q4'])

Date features

df['date'] = pd.to_datetime(df['date'])
df['year']        = df['date'].dt.year
df['month']       = df['date'].dt.month
df['day_of_week'] = df['date'].dt.dayofweek   # 0=Monday
df['is_weekend']  = df['day_of_week'] >= 5
df['quarter']     = df['date'].dt.quarter
df['days_to_end_of_month'] = (df['date'] + pd.offsets.MonthEnd(0) - df['date']).dt.days

Derived ratios and interactions

df['revenue_per_hour']  = df['sales'] / df['hours_worked'].replace(0, np.nan)
df['deal_size']         = df['sales'] / df['deals_closed'].replace(0, np.nan)
df['sales_x_experience'] = df['sales'] * df['years_experience']  # interaction

# Log transform (normalises right-skewed distributions)
df['log_sales'] = np.log1p(df['sales'])   # log1p handles 0s safely

Full cleaning pipeline

import pandas as pd, numpy as np
from pathlib import Path

def clean_sales(path: str) -> pd.DataFrame:
    df = pd.read_csv(path)

    # 1. Standardise column names
    df.columns = df.columns.str.strip().str.lower().str.replace(' ', '_')

    # 2. Drop rows missing key identifier
    df = df.dropna(subset=['sale_id'])

    # 3. Types
    df['date']  = pd.to_datetime(df['date'], dayfirst=True, errors='coerce')
    df['sales'] = pd.to_numeric(df['sales'].astype(str).str.replace(r'[$,]','',regex=True), errors='coerce')

    # 4. String normalisation
    df['region'] = df['region'].str.strip().str.title()
    df['email']  = df['email'].str.strip().str.lower()

    # 5. Fill missing values
    df['region'].fillna('Unknown', inplace=True)
    df['sales'].fillna(df.groupby('region')['sales'].transform('median'), inplace=True)

    # 6. Remove outliers (IQR)
    Q1, Q3 = df['sales'].quantile([0.25, 0.75])
    IQR = Q3 - Q1
    df = df[(df['sales'] >= Q1 - 1.5*IQR) & (df['sales'] <= Q3 + 1.5*IQR)]

    # 7. Dedup
    df = df.drop_duplicates(subset=['sale_id']).reset_index(drop=True)

    # 8. Feature engineering
    df['month']      = df['date'].dt.month
    df['quarter']    = df['date'].dt.quarter
    df['is_weekend'] = df['date'].dt.dayofweek >= 5

    print(f"Clean: {len(df)} rows, {df.shape[1]} columns")
    return df

df_clean = clean_sales('raw_sales.csv')
df_clean.to_csv('clean_sales.csv', index=False)
