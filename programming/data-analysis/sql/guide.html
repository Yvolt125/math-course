Chapter 1
SELECT Fundamentals

SQL (Structured Query Language) is the standard language for working with relational databases. As a data analyst you'll write more SELECT queries than anything else.

Basic query anatomy

SELECT  rep_name, region, sales         -- which columns
FROM    sales                            -- which table
WHERE   sales > 10000                    -- filter rows
ORDER BY sales DESC                      -- sort result
LIMIT   10;                              -- take first 10 rows

Result

rep_nameregionsales
DaveEast18300
BobSouth15800
EveSouth14200
AliceNorth12400

Selecting columns

SELECT *                          -- all columns (avoid in production)
SELECT rep_name, sales
SELECT rep_name AS name,
       sales * 0.08 AS commission  -- aliasing and expressions
SELECT DISTINCT region            -- unique values only
SQL execution order SQL clauses run in this order, regardless of how you write them:
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
This matters: you cannot use a SELECT alias in a WHERE clause because WHERE runs before SELECT.

Data types

-- SQLite (most common for local analysis)
TEXT            -- any string
INTEGER         -- whole numbers
REAL            -- floating-point
BLOB            -- binary data
NULL            -- missing value

-- Type casting
CAST(sales AS REAL)
CAST(date_str AS DATE)
Chapter 2
Filtering & Sorting
-- Comparisons
WHERE sales > 10000
WHERE sales BETWEEN 10000 AND 20000
WHERE region = 'South'
WHERE region != 'North'

-- NULL checks
WHERE email IS NULL
WHERE email IS NOT NULL

-- List membership
WHERE region IN ('North', 'East')
WHERE region NOT IN ('Unknown', 'N/A')

-- Pattern matching
WHERE rep_name LIKE 'A%'        -- starts with A
WHERE email LIKE '%@gmail.com'  -- ends with @gmail.com
WHERE code LIKE 'A_B'           -- _ matches any single char

-- Logical operators
WHERE region = 'South' AND sales > 12000
WHERE month = 'Jan'   OR  month = 'Feb'
WHERE NOT (region = 'North')

Sorting

ORDER BY sales DESC           -- highest first
ORDER BY region, sales DESC  -- by region first, then sales descending
ORDER BY 3 DESC               -- sort by 3rd column in SELECT list

Date filtering

-- In SQLite dates are stored as strings: 'YYYY-MM-DD'
WHERE date >= '2024-01-01'
WHERE date BETWEEN '2024-01-01' AND '2024-03-31'
WHERE strftime('%Y', date) = '2024'
WHERE strftime('%m', date) = '01'  -- January only
Chapter 3
Aggregation & GroupBy
-- Aggregate functions
SELECT
    COUNT(*)          AS total_rows,
    COUNT(email)      AS non_null_emails,
    SUM(sales)        AS total_sales,
    AVG(sales)        AS avg_sales,
    MAX(sales)        AS highest_sale,
    MIN(sales)        AS lowest_sale
FROM sales;

GROUP BY

SELECT
    region,
    SUM(sales)   AS total_sales,
    COUNT(*)     AS num_deals,
    ROUND(AVG(sales), 2) AS avg_deal
FROM sales
GROUP BY region
ORDER BY total_sales DESC;

HAVING — filter on aggregated values

HAVING is like WHERE but applied after aggregation. Use it to filter groups, not individual rows.

SELECT region, SUM(sales) AS total
FROM   sales
GROUP BY region
HAVING SUM(sales) > 20000    -- only groups with total > 20000
ORDER BY total DESC;

-- Multi-level grouping
SELECT region, month, SUM(sales) AS total
FROM   sales
GROUP BY region, month
ORDER BY region, month;
Chapter 4
Joins

Joins combine rows from two tables using a shared key.

INNER JOIN
Only rows that match in BOTH tables.
Most common. Drops non-matching rows.
LEFT JOIN
All rows from left table; NULLs for non-matching right rows.
Use when left table is the "source of truth".
RIGHT JOIN
All rows from right table; NULLs for non-matching left rows.
Rare — can always rewrite as LEFT JOIN.
FULL OUTER JOIN
All rows from both tables; NULLs where no match.
Not supported in SQLite directly.
-- INNER JOIN: only matching reps
SELECT s.rep_id, r.name, r.region, s.sales
FROM   sales   s
JOIN   reps    r ON s.rep_id = r.rep_id;

-- LEFT JOIN: all sales, even if rep details missing
SELECT s.sale_id, r.name, s.sales
FROM   sales s
LEFT JOIN reps r ON s.rep_id = r.rep_id;

-- Find rows with NO match (anti-join pattern)
SELECT s.*
FROM   sales s
LEFT JOIN reps r ON s.rep_id = r.rep_id
WHERE  r.rep_id IS NULL;    -- sales with unknown rep

-- Three-table join
SELECT s.sale_id, r.name, p.product_name, s.amount
FROM   sales s
JOIN   reps     r ON s.rep_id     = r.rep_id
JOIN   products p ON s.product_id  = p.product_id;
Always use table aliases s, r, p are aliases. Without them, sales.rep_id is explicit but verbose. Aliases make multi-join queries readable.
Chapter 5
Subqueries & CTEs

Subqueries

A subquery is a query nested inside another query.

-- In WHERE: find reps who made more than average
SELECT rep_name, sales
FROM   sales
WHERE  sales > (SELECT AVG(sales) FROM sales);

-- In FROM: treat aggregate result as a table
SELECT t.region, t.total, t.total / grand.grand_total * 100 AS pct
FROM (
    SELECT region, SUM(sales) AS total
    FROM   sales
    GROUP BY region
) t,
(SELECT SUM(sales) AS grand_total FROM sales) grand;

CTEs — Common Table Expressions

CTEs (WITH clauses) give subqueries a name. They make complex queries far easier to read.

WITH regional_totals AS (
    SELECT region, SUM(sales) AS total
    FROM   sales
    GROUP BY region
),
grand_total AS (
    SELECT SUM(total) AS grand
    FROM   regional_totals
)
SELECT
    rt.region,
    rt.total,
    ROUND(rt.total * 100.0 / gt.grand, 1) AS pct_of_total
FROM  regional_totals rt, grand_total gt
ORDER BY rt.total DESC;
CTEs over subqueries Prefer CTEs whenever a subquery is used more than once or the query is getting long. They are easier to debug — you can run just the CTE part to check its output.
Chapter 6
Window Functions

Window functions compute a value over a window of rows related to the current row — without collapsing rows like GROUP BY does. They are among the most powerful tools in SQL.

-- Syntax: function() OVER (PARTITION BY ... ORDER BY ...)

-- Running total
SELECT
    date,
    sales,
    SUM(sales) OVER (ORDER BY date) AS running_total
FROM sales;

-- Rank within each region
SELECT
    rep_name, region, sales,
    RANK()     OVER (PARTITION BY region ORDER BY sales DESC) AS rnk,
    ROW_NUMBER() OVER (PARTITION BY region ORDER BY sales DESC) AS row_num,
    DENSE_RANK() OVER (PARTITION BY region ORDER BY sales DESC) AS dense_rnk
FROM sales;

-- Each rep's share of their region's total
SELECT
    rep_name, region, sales,
    SUM(sales) OVER (PARTITION BY region) AS region_total,
    ROUND(sales * 100.0 / SUM(sales) OVER (PARTITION BY region), 1) AS share_pct
FROM sales;

-- Moving average (7-day)
SELECT
    date, sales,
    AVG(sales) OVER (
        ORDER BY date
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) AS moving_avg_7d
FROM daily_sales;

-- Previous row value (LAG)
SELECT
    month, sales,
    LAG(sales, 1, 0) OVER (ORDER BY month) AS prev_month_sales,
    sales - LAG(sales, 1, 0) OVER (ORDER BY month) AS month_over_month
FROM monthly_sales;
Chapter 7
Writing & Modifying Data
-- Create table
CREATE TABLE sales (
    sale_id  INTEGER PRIMARY KEY AUTOINCREMENT,
    rep_id   INTEGER NOT NULL,
    sales    REAL    NOT NULL,
    month    TEXT,
    date     TEXT,
    FOREIGN KEY (rep_id) REFERENCES reps(rep_id)
);

-- Insert rows
INSERT INTO sales (rep_id, sales, month, date)
VALUES (101, 12400, 'Jan', '2024-01-15');

-- Insert from another table
INSERT INTO sales_archive
SELECT * FROM sales WHERE date < '2023-01-01';

-- Update
UPDATE sales
SET    sales = sales * 1.05
WHERE  region = 'North' AND month = 'Jan';

-- Delete
DELETE FROM sales WHERE sale_id = 42;

-- Add a column
ALTER TABLE sales ADD COLUMN commission REAL;
Always test UPDATE/DELETE with SELECT first Replace UPDATE sales SET ... WHERE ... with SELECT * FROM sales WHERE ... to verify you have the right rows before modifying them.
Chapter 8
Python Integration

The most common workflow: run SQL from Python, load results directly into a Pandas DataFrame.

sqlite3 — built-in, no setup

import sqlite3, pandas as pd

conn = sqlite3.connect('company.db')    # creates file if it doesn't exist

# Execute a query → DataFrame
df = pd.read_sql("""
    SELECT region, SUM(sales) AS total
    FROM   sales
    GROUP  BY region
    ORDER  BY total DESC
""", conn)

print(df)

# Parameterised query (safe — never build SQL by string formatting)
region = 'South'
df = pd.read_sql(
    "SELECT * FROM sales WHERE region = ?",
    conn,
    params=(region,)
)

conn.close()

Write a DataFrame to SQLite

conn = sqlite3.connect('company.db')

df.to_sql(
    'sales_cleaned',    # table name
    conn,
    if_exists='replace',  # 'replace', 'append', or 'fail'
    index=False
)

conn.close()

Run arbitrary SQL (INSERT / CREATE / etc.)

conn = sqlite3.connect('company.db')
cur  = conn.cursor()

cur.execute("""
    CREATE TABLE IF NOT EXISTS reps (
        rep_id INTEGER PRIMARY KEY,
        name   TEXT NOT NULL,
        region TEXT
    )
""")

# Insert multiple rows from Python list
reps_data = [(101,'Alice','North'), (102,'Bob','South'), (103,'Carol','North')]
cur.executemany("INSERT OR IGNORE INTO reps VALUES (?,?,?)", reps_data)

conn.commit()
conn.close()

pandas + SQL is a superpower Use SQL to filter and aggregate the data at the database level (fast, efficient), then load only what you need into pandas for Python-specific transformations and plotting. Avoid loading entire tables into memory when a WHERE clause can cut the data down first.

Complete workflow — SQL + Pandas pipeline

import sqlite3, pandas as pd
import matplotlib.pyplot as plt
from pathlib import Path

# Connect
conn = sqlite3.connect('sales.db')

# Aggregate in SQL
df = pd.read_sql("""
    WITH monthly AS (
        SELECT strftime('%Y-%m', date) AS month,
               region,
               SUM(sales)             AS total
        FROM   sales
        GROUP  BY month, region
    )
    SELECT *
    FROM   monthly
    ORDER  BY month, region
""", conn)
conn.close()

# Pivot for plotting
pivot = df.pivot(index='month', columns='region', values='total').fillna(0)

# Plot
fig, ax = plt.subplots(figsize=(12, 5))
for col in pivot.columns:
    ax.plot(pivot.index, pivot[col], marker='o', label=col, linewidth=2)
ax.set_title('Monthly Sales by Region')
ax.set_xlabel('Month')
ax.set_ylabel('Total Sales ($)')
ax.legend()
ax.tick_params(axis='x', rotation=45)
ax.yaxis.set_major_formatter(plt.FuncFormatter(lambda x,_: f'${x/1e3:.0f}k'))
ax.spines[['top','right']].set_visible(False)
plt.tight_layout()
plt.savefig('monthly_trend.png', dpi=150)
print("Done.")
