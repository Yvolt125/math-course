Chapter 1
The File System
What it is · The tree · Files vs directories

What is a file system?

When you save a file, Python does not reach directly into the spinning magnetic disk or SSD flash cells. There is a layer in between — the file system — whose job is to organise raw storage into named, accessible units. It keeps track of where every file lives, how big it is, when it was last modified, and who is allowed to read or write it.

The os module is Python's gateway to the file system. It provides a uniform interface across Windows, Mac, and Linux — you write one script, it works everywhere.

The core idea: the file system is a phone book for your disk. A filename is the name you look up; the file system tells the OS where the actual bytes live.

The tree structure

Every file system is organised as an inverted tree. There is one root at the top, and everything branches downward from it. Directories (folders) are the branches; files are the leaves.

/ (root) ├── home/ (directory) │ └── alice/ │ ├── projects/ │ │ ├── analyse.py (file) │ │ └── data/ │ │ ├── sales.csv │ │ └── costs.csv │ └── notes.txt (file) ├── etc/ └── tmp/

Purple = directory  ·  grey = file  ·  root is / on Linux/Mac, C:\ on Windows

A directory is nothing magical — it is itself stored as a small file that contains a list of names and pointers to its children. A file is a named sequence of bytes. That's it.

Files vs directories at a glance

Can be empty?Yes (0 bytes)Yes (no children)
Has extension?Usually (hint only)Rarely (no rule)

Linux/Mac vs Windows paths

The two path conventions you'll encounter constantly:

Linux / Mac /home/alice/data/sales.csv Windows C:\Users\alice\data\sales.csv

Linux/Mac uses / as separator, root is /  ·  Windows uses \, root is a drive letter like C:\

Good news: if you use os.path.join() or pathlib.Path to build paths (never string concatenation), Python picks the right separator for you automatically.

File extensions

An extension like .csv or .py is just the last part of the filename after the final dot. It is a convention, not enforced by the OS. You can rename data.csv to data.banana — the bytes inside are unchanged. Python programs use os.path.splitext() or Path.suffix to read the extension so they can decide how to process a file.

Chapter 2
Navigating the File System
import os · getcwd · chdir · absolute vs relative

Importing os

Everything in this guide starts with one line:

import os

The os module is part of Python's standard library — no installation needed. It provides hundreds of functions for interacting with the operating system: file paths, environment variables, process management, and more.

The current working directory (cwd)

Every running process has a current working directory — the folder it is currently "standing in." Think of it as your GPS position in the file-system tree. All relative paths are calculated from this position.

/home/alice/ ├── projects/ ← cwd (you are here) │ └── analyse.py └── data/ └── sales.csv
From cwd /home/alice/projects/ "../data/sales.csv" → /home/alice/data/sales.csv ✓ "sales.csv" → /home/alice/projects/sales.csv 
import os

# Where are we right now?
cwd = os.getcwd()
print(cwd)
# /home/alice/projects

# Change directory
os.chdir("/home/alice/data")
print(os.getcwd())
# /home/alice/data
Warning: os.chdir() changes the cwd for the entire Python process from that point onward. Any relative path used later will resolve differently. Prefer building absolute paths instead of relying on chdir — your script will be much more predictable.

Absolute vs relative paths

This distinction is one of the most important concepts in working with files.

Absolute pathRelative path
Starts with/ or C:\anything else
Resolves fromThe root — always the sameThe current working directory
Example/home/alice/data/sales.csvdata/sales.csv
Safe to use when…Always — never ambiguousOnly if you control the cwd

Special path symbols

.
Current directory. ./script.py = script.py in the cwd.
..
Parent directory. ../data = the data folder one level up from cwd.
~
Home directory of the current user. Python: os.path.expanduser("~").

Building an absolute path from a relative one

import os

# Convert relative → absolute (based on current cwd)
abs_path = os.path.abspath("data/sales.csv")
print(abs_path)
# /home/alice/projects/data/sales.csv

# Expand ~ to the real home path
home = os.path.expanduser("~")
print(home)
# /home/alice
Best practice for scripts: anchor paths relative to the script file itself, not the cwd. This way the script works no matter where you run it from:

SCRIPT_DIR = os.path.dirname(os.path.abspath(__file__))
DATA_FILE = os.path.join(SCRIPT_DIR, "data", "sales.csv")

Chapter 3
Listing & Finding Files
os.listdir · os.walk · glob · rglob

os.listdir() — look inside one folder

os.listdir(path) returns a plain Python list of names (not full paths) of everything inside the given directory — files and subdirectories alike. It does not recurse into subfolders.

import os

names = os.listdir("/home/alice/data")
print(names)
# ['sales.csv', 'costs.csv', 'archive', '.DS_Store']

# To get full paths, you must join manually:
folder = "/home/alice/data"
full_paths = [os.path.join(folder, name) for name in os.listdir(folder)]
# ['/home/alice/data/sales.csv', '/home/alice/data/costs.csv', ...]
Note: os.listdir() includes hidden files (those starting with . on Linux/Mac), and returns items in arbitrary order. Use sorted(os.listdir(path)) for consistent ordering.

os.walk() — traverse the entire tree

When you need to process files in subdirectories too, os.walk() is the tool. It walks down the tree and at each directory yields a 3-tuple:

dirpath
The current directory as a string, e.g. "/home/alice/data/2024"
dirnames
List of subdirectory names inside dirpath (bare names, not full paths)
filenames
List of file names inside dirpath (bare names, not full paths)
Directory tree being walked: /home/alice/data/ ├── sales.csv ├── costs.csv └── 2024/ ├── q1.csv └── q2.csv os.walk yields these tuples: (dirpath, dirnames, filenames) ("/home/alice/data", ["2024"], ["sales.csv","costs.csv"]) ("/home/alice/data/2024", [], ["q1.csv","q2.csv"]) 
import os

for dirpath, dirnames, filenames in os.walk("/home/alice/data"):
    for fname in filenames:
        if fname.endswith(".csv"):
            full = os.path.join(dirpath, fname)
            print(full)

# /home/alice/data/sales.csv
# /home/alice/data/costs.csv
# /home/alice/data/2024/q1.csv
# /home/alice/data/2024/q2.csv

glob — find files with wildcards

The glob module lets you find files using wildcard patterns — much more expressive than filtering a list manually.

*
Matches any sequence of characters within one directory level
?
Matches exactly one character
**
Matches zero or more directory levels (use with recursive=True)
import glob

# All CSVs in one folder
files = glob.glob("/home/alice/data/*.csv")
# ['/home/alice/data/sales.csv', '/home/alice/data/costs.csv']

# All CSVs anywhere under /data (recursive)
files = glob.glob("/home/alice/data/**/*.csv", recursive=True)
# ['/home/alice/data/sales.csv', ..., '/home/alice/data/2024/q1.csv', ...]

# Always sort for reproducibility
files = sorted(glob.glob("/home/alice/data/**/*.csv", recursive=True))

Choosing the right tool

ToolUse when…Returns
os.listdir()You want everything in one folderList of bare names
os.walk()You need to recurse and also know the directory path at each stepGenerator of (dirpath, dirs, files)
glob.glob()You want files matching a wildcard pattern, optionally recursiveList of path strings

Chapter 4
Path Tools
os.path · pathlib.Path · Building output paths

os.path — the classic approach

os.path is a sub-module of os containing functions that operate on path strings. It is the traditional way to work with paths and you will see it everywhere in older codebases.

path = "/home/alice/data/sales_2024.csv" Breaking it apart: dirname(path) → "/home/alice/data" basename(path) → "sales_2024.csv" splitext(path) → ("/home/alice/data/sales_2024", ".csv") Checking: exists(path) → True / False isfile(path) → True / False isdir(path) → True / False getsize(path) → size in bytes 
import os

path = "/home/alice/data/sales_2024.csv"

# Break apart a path
folder = os.path.dirname(path)      # "/home/alice/data"
name   = os.path.basename(path)     # "sales_2024.csv"
stem, ext = os.path.splitext(name)  # ("sales_2024", ".csv")

# Build a new path (always join, never concatenate)
output = os.path.join(folder, stem + "_cleaned" + ext)
# "/home/alice/data/sales_2024_cleaned.csv"

# Safety checks
if os.path.exists(path):
    size_mb = os.path.getsize(path) / 1024**2
    print(f"File is {size_mb:.1f} MB")
Common gotcha with os.path.join(): if any argument is an absolute path, everything before it is thrown away.
os.path.join("/data", "/output/file.csv") → "/output/file.csv" — not what you intended!

pathlib — the modern approach

Introduced in Python 3.4, pathlib represents paths as objects rather than strings. The same operations become cleaner and more readable. Prefer pathlib for new code.

from pathlib import Path

p = Path("/home/alice/data/sales_2024.csv")

# Attributes — no function calls needed
p.parent   # Path("/home/alice/data")
p.name     # "sales_2024.csv"
p.stem     # "sales_2024"
p.suffix   # ".csv"

# Joining with the / operator
output_dir = Path("/home/alice/output")
output_file = output_dir / p.name
# Path("/home/alice/output/sales_2024.csv")

# Swap the extension
parquet = p.with_suffix(".parquet")
# Path("/home/alice/data/sales_2024.parquet")

# Swap the filename entirely
summary = p.with_name("summary.csv")
# Path("/home/alice/data/summary.csv")

# Check existence
p.exists()    # True/False
p.is_file()  # True/False
p.is_dir()   # True/False

Creating directories safely

from pathlib import Path

output = Path("/home/alice/output/2024/q1")

# Create the full path including any missing parents
# exist_ok=True means no error if it already exists
output.mkdir(parents=True, exist_ok=True)

# os equivalent:
import os
os.makedirs("/home/alice/output/2024/q1", exist_ok=True)

Mirroring a directory structure

A very common pattern in data pipelines: read from data/raw/, write to data/processed/ preserving the same subfolder layout.

Input: Output: data/raw/ data/processed/ ├── 2024/ ├── 2024/ │ ├── q1/ │ ├── q1/ │ │ └── jan.csv →│ │ └── jan.csv │ └── q2/ │ └── q2/ │ └── apr.csv →│ └── apr.csv └── 2023/ └── 2023/ └── full.csv → └── full.csv 
from pathlib import Path

input_root  = Path("data/raw")
output_root = Path("data/processed")

for src in input_root.rglob("*.csv"):
    # e.g. src = Path("data/raw/2024/q1/jan.csv")
    rel      = src.relative_to(input_root)   # Path("2024/q1/jan.csv")
    dest     = output_root / rel               # Path("data/processed/2024/q1/jan.csv")
    dest.parent.mkdir(parents=True, exist_ok=True)
    # ... process src, write to dest

Chapter 5
Reading & Writing Files
Text · CSV · JSON

The open() function and file modes

All file I/O in Python goes through open(path, mode). The mode controls what you're allowed to do:

"r"
Read (default). File must exist.
"w"
Write. Creates file if missing, truncates (erases) if it exists.
"a"
Append. Creates if missing, writes to the end if it exists.
"x"
Exclusive create. Fails if file already exists — safe for new files.
"b"
Binary mode — add to any of the above: "rb", "wb", etc.

Always use the with statement

The with statement is a context manager. It guarantees the file is closed when the block exits — even if an exception is raised. Never skip it.

# ✓ Correct — file closes automatically
with open("notes.txt", "r", encoding="utf-8") as f:
    content = f.read()   # entire file as one string

# ✗ Risky — if an error occurs, f.close() is never called
f = open("notes.txt")
content = f.read()
f.close()

Reading methods compared

MethodReturnsBest for
f.read()Entire file as one stringSmall files you need in memory at once
f.readlines()List of lines (with \n)When you need all lines as a list
for line in f:One line at a time (iterator)Large files — reads only one line into memory at a time
# Memory-efficient line-by-line processing
with open("big_log.txt", encoding="utf-8") as f:
    for line in f:
        line = line.strip()   # removes \n and surrounding whitespace
        if "ERROR" in line:
            print(line)

CSV files — the csv module

Never split CSV lines by comma manually — a value like "Smith, John" contains a comma inside quotes and would be broken by naive splitting. The csv module handles all edge cases.

Sample file: data.csv name,amount,category Alice,1200.00,salary Bob,"2,500.00",bonus ← comma inside quotes — naive split breaks this Carol,800.00,freelance 
import csv

# Reading with DictReader — access fields by name, not index
with open("data.csv", newline="", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        # row is a dict: {"name": "Alice", "amount": "1200.00", ...}
        name   = row["name"]
        amount = float(row["amount"])  # always cast — csv gives strings
        print(f"{name}: £{amount:.2f}")

# Writing with DictWriter
rows = [
    {"name": "Alice", "total": 1200},
    {"name": "Bob",   "total": 2500},
]
with open("output.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "total"])
    writer.writeheader()
    writer.writerows(rows)
Always pass newline="" when opening CSV files. Without it, Python's newline translation can add extra blank rows — especially on Windows.

JSON files — the json module

JSON maps naturally to Python dicts and lists. The four functions you need:

json.load(f)
Parse JSON from an open file → Python object
json.loads(s)
Parse JSON from a string → Python object
json.dump(obj, f)
Write Python object as JSON to an open file
json.dumps(obj)
Convert Python object to a JSON string
import json

# Reading a JSON config file
with open("config.json", encoding="utf-8") as f:
    config = json.load(f)
    # config is now a Python dict

timeout = config.get("timeout", 30)  # fallback to 30 if key missing

# Writing — indent=4 makes it human-readable
results = {"processed": 142, "errors": 3, "skipped": 58}
with open("results.json", "w", encoding="utf-8") as f:
    json.dump(results, f, indent=4)
What json cannot handle: datetime, set, bytes, and custom objects all raise TypeError. Convert datetimes to .isoformat() strings first. Sets to lists.

What are environment variables?

Every process running on your computer has a set of environment variables — a dictionary of key-value string pairs provided by the operating system. They are used to pass configuration to programs without hardcoding values in source code.

OS Environment (your process's copy): HOME = /home/alice PATH = /usr/local/bin:/usr/bin:/bin LANG = en_GB.UTF-8 DATABASE_URL = postgres://localhost:5432/mydb ← you set this API_KEY = sk-abc123xyz ← you set this 
import os

# Read a variable — raises KeyError if missing
db_url = os.environ["DATABASE_URL"]

# Read with a fallback — safe for optional config
timeout = os.environ.get("TIMEOUT", "30")
timeout = int(timeout)   # env vars are ALWAYS strings — cast explicitly

# List all environment variables
for key, value in os.environ.items():
    print(f"{key}={value}")

Why env vars for secrets?

The golden rule: never put API keys, passwords, or tokens in your source code. Once committed to git, a secret lives in history forever — even after you delete it. Environment variables stay outside the codebase entirely.

The .env pattern with python-dotenv

# .env file (never commit this to git)
DATABASE_URL=postgres://localhost:5432/mydb
API_KEY=sk-abc123xyz

# In your Python script
from dotenv import load_dotenv
import os

load_dotenv()   # reads .env and injects into os.environ

api_key = os.environ["API_KEY"]   # now works

subprocess — running shell commands from Python

Sometimes you need to call an external program (git, ffmpeg, a CLI tool). subprocess.run() is the modern, safe way to do this.

import subprocess

# Run a command and wait for it to finish
result = subprocess.run(
    ["git", "log", "--oneline", "-5"],
    capture_output=True,   # capture stdout and stderr
    text=True,              # decode bytes to string automatically
    check=True              # raise exception if command fails
)

print(result.stdout)
print("Exit code:", result.returncode)  # 0 = success
Always pass a list, not a string. ["git", "log"] is safe — each element is a separate argument. shell=True with a string is dangerous if any part of the command comes from user input — it opens the door to shell injection attacks.
ParameterWhat it does
capture_output=TrueCaptures stdout and stderr instead of printing them to the terminal
text=TrueDecodes output as a string (instead of bytes)
check=TrueRaises CalledProcessError if exit code is non-zero
env={...}Pass a custom environment dict to the child process
cwd="path"Set the working directory for the child process

Automation Scripts
shutil · Batch processing · Best practices

shutil — high-level file operations

The os module handles low-level operations (rename, delete one file). For copying, moving, and deleting entire directory trees, use shutil (shell utilities).

shutil.copy(src, dst)
Copy file content + permissions. Timestamps are not preserved.
shutil.copy2(src, dst)
Copy file content + permissions + timestamps. Use for exact duplicates.
shutil.copytree(src, dst)
Recursively copy an entire directory tree.
shutil.move(src, dst)
Move or rename. Works across filesystems (unlike os.rename).
shutil.rmtree(path)
Delete a directory and everything inside it. Irreversible.
shutil.disk_usage(path)
Returns (total, used, free) in bytes for the disk at path.
import shutil, os

# Copy a file (preserving metadata)
shutil.copy2("report.csv", "/backup/report.csv")

# Move a file
shutil.move("temp/output.csv", "final/output.csv")

# Safe delete — check first!
path = "/tmp/old_results"
if os.path.exists(path):
    shutil.rmtree(path)

# Check available disk space before writing
usage = shutil.disk_usage("/")
free_gb = usage.free / 1024**3
print(f"{free_gb:.1f} GB free")

The batch processing pattern

Almost every data automation task follows the same four-step pattern. Knowing it as a named pattern makes scripts easier to write and explain.

1 · Scan
→
2 · Filter
→
3 · Process
→
4 · Write
import os, shutil, glob
from pathlib import Path

INPUT_DIR  = Path("data/raw")
OUTPUT_DIR = Path("data/processed")
OUTPUT_DIR.mkdir(parents=True, exist_ok=True)

moved = skipped = errors = 0

# ① SCAN — find all input files
all_files = sorted(INPUT_DIR.rglob("*.csv"))

for src in all_files:

    # ② FILTER — skip already-processed files (idempotency)
    dest = OUTPUT_DIR / src.relative_to(INPUT_DIR)
    if dest.exists():
        skipped += 1
        continue

    # ③ PROCESS — your logic here
    try:
        dest.parent.mkdir(parents=True, exist_ok=True)
        # (example: just copy — replace with real processing)
        shutil.copy2(src, dest)
        print(f"  ✓ {src.name}")
        moved += 1

    # ④ WRITE — errors are logged, not fatal
    except Exception as e:
        print(f"  ✗ {src.name}: {e}")
        errors += 1

print(f"\nDone: {moved} processed, {skipped} skipped, {errors} errors")

Making scripts robust — best practices

Idempotency: a script is idempotent if running it twice gives the same result as running it once. Check whether the output already exists before processing. This makes scripts safe to re-run after crashes.
Atomic writes: write to a temp file first, then rename when done. If the script crashes mid-write, you never end up with a corrupt half-written output file.

with open("output.csv.tmp", "w") as f: ...
os.rename("output.csv.tmp", "output.csv")
Never overwrite originals: always write to a separate output location. Only delete or overwrite source data after you have verified the output is correct.
