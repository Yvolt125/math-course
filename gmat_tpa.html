Lesson 1

Paired Answers

Objectives
  • Understand that TPA requires selecting two answers — one for each part — from a single shared answer column.
  • Recognise that the two selections are interdependent and must together satisfy a combined condition.
  • Work efficiently by fixing one part and solving for the other algebraically.
📌 Core Concept

Two-Part Analysis presents a problem and a table with three columns: two "Select" columns (one for each part) and a shared answer column. You choose one row for Part 1 and a different row for Part 2 — they cannot be the same row.

The two selections together must satisfy the question's combined requirement. The task is not to evaluate each part independently but to find the pair that works as a system.

How to Approach It
  1. 1Identify what each Part is asking — they are usually linked by a constraint (e.g., "Part 1 + Part 2 = target").
  2. 2Fix one variable and solve for the other algebraically before looking at the answer table.
  3. 3Check whether both values appear in the answer column.
  4. 4Verify that the pair together satisfies the original combined constraint.
Example

A project budget of $120,000 must be allocated between two departments. Marketing must receive twice as much as Operations. In the table below, select the amount allocated to Marketing (Part 1) and the amount allocated to Operations (Part 2).

Part 1 — MarketingPart 2 — OperationsAmount
○○$20,000
○○$40,000
○○$60,000
○○$80,000
○○$100,000
▶ Show Answer

Answer: Marketing = $80,000 (Part 1); Operations = $40,000 (Part 2). Let Operations = x; Marketing = 2x. Together: 2x + x = $120,000 → x = $40,000. Marketing = $80,000. Both values appear in the answer column; they must be different rows. Common trap: selecting $60,000 for both (same row), which violates the rule — and 60 + 60 ≠ 120 in the correct ratio.

Common Traps
  • Selecting the same row for both parts — the rules require different rows for each part.
  • Solving for one part correctly but choosing the wrong row because you misread the shared answer column.
Summary
  • Two selections from different rows of the same answer column — they form a pair.
  • The selections are interdependent — one determines the other via a shared constraint.
  • Fix Part 1, solve for Part 2, then verify both values appear in the answer column.
💡 Quick Tip

Treat TPA like a system of two equations. Name the two unknowns, write the constraint linking them, and solve algebraically before looking at the answer table — this avoids trial-and-error guessing.

Lesson 2

Logical + Quantitative Combo

Objectives
  • Handle TPA questions that mix a logical condition with a quantitative one.
  • Identify which part is constrained logically and which requires calculation.
  • Eliminate impossible answer pairs before solving the remaining constraint.
📌 Core Concept

Many TPA questions combine a verbal/logical constraint (e.g., "must be consistent with the argument") with a mathematical constraint (e.g., "the values must sum to a target"). The structure — two conditions, one answer table, select one per condition — stays constant regardless of content type.

Resolve the logical part first to narrow candidates, then apply the quantitative condition to find the valid pair. Doing it in the wrong order wastes time computing solutions that are logically invalid.

How to Approach It
  1. 1Classify each part as primarily logical or primarily quantitative.
  2. 2Apply the logical constraint first to eliminate rows from consideration.
  3. 3Among remaining options, apply the quantitative condition to find the valid pair.
  4. 4Verify both parts satisfy their respective constraints simultaneously.
Example

A company sells two products, X and Y. Products sold at a loss cannot be selected for the "expand" list. Product X has a margin of −5%; Product Y has a margin of +12%; Product Z has a margin of +3%. The total units sold for the two selected products must equal 500. Units sold: X = 200, Y = 300, Z = 500. Select one product to expand (Part 1) and one product whose units equal the remaining target (Part 2) so the total = 500.

Part 1 — ExpandPart 2 — RemainingUnits Sold
○○200 (Product X)
○○300 (Product Y)
○○500 (Product Z)
▶ Show Answer

Answer: Part 1 = Product Y (300 units); Part 2 = Product X (200 units). Logical constraint first: Product X has a negative margin (−5%) so it cannot be expanded — eliminate X from Part 1. Remaining Part 1 candidates: Y (300) or Z (500). Quantitative constraint: Part 1 + Part 2 = 500. If Part 1 = Y (300) → Part 2 must = 200 (Product X). ✓ If Part 1 = Z (500) → Part 2 must = 0 (not in the table). ✗ Answer: expand Y (300), pair with X (200).

Common Traps
  • Applying the quantitative constraint before reading the logical one — this wastes time solving for pairs that are logically invalid from the start.
  • Ignoring that the two parts interact — a value valid for Part 1 alone may become invalid when paired with the Part 2 selection.
Summary
  • Logical constraint first — eliminate invalid options before computing anything.
  • Then quantitative constraint — find the valid pair among remaining options.
  • Verify both parts together at the end to confirm the pair is mutually consistent.
💡 Quick Tip

In mixed TPA questions, scan the logical constraint for "must," "cannot," or "always/never" language — these give you hard exclusions before you compute anything, often reducing candidates to just 2–3 rows.
