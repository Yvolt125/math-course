Section 1 — Derivatives: The Rate of Change Tangent line · Power rule · Key derivatives

The derivative of \(f\) at \(x\) measures how fast \(f\) is changing at that exact moment — the slope of the tangent line to the curve.

Limit Definition
\[ f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h} \]
Power Rule
\[ \frac{d}{dx}\left[x^n\right] = n \cdot x^{n-1} \] Bring the exponent down, reduce the power by 1.

Key Rules

  • Sum rule: \((f + g)' = f' + g'\)
  • Constant multiple: \((cf)' = c \cdot f'\)
  • Product rule: \((fg)' = f'g + fg'\)
  • Quotient rule: \(\left(\tfrac{f}{g}\right)' = \tfrac{f'g - fg'}{g^2}\)

Essential Derivatives for ML

FunctionDerivativeWhy it matters
\(e^x\)\(e^x\)Self-derivative — appears in softmax, cross-entropy, sigmoid

Worked Example
\(f(x) = 3x^2 + 2x - 5\). Find \(f'(x)\) and evaluate at \(x=1\).

\(f'(x) = 6x + 2\). At \(x=1\): \(f'(1) = 8\).
ML Connection
In gradient descent we compute \(\partial L / \partial w\). If positive, increasing \(w\) makes the loss worse — we decrease \(w\). The derivative is the compass; we walk opposite to it.

Section 2 — Optimisation & Gradient Descent Update rule · Learning rate · SGD · Convergence

Training a model is an optimisation problem: find weights \(\mathbf{w}\) minimising loss \(L(\mathbf{w})\). At a minimum, \(L'(w^*)=0\) and \(L''(w^*)>0\).

Gradient Descent Update Rule
\[ w \leftarrow w - \alpha \cdot \frac{\partial L}{\partial w} \] \(\alpha\) = learning rate. Gradient points uphill; subtracting it goes downhill.
3 Steps on L(w) = w², w₀ = 4, α = 0.1
Gradient: \(\partial L/\partial w = 2w\)

Step 1: \(w = 4 - 0.1 \times 8 = 3.2\)
Step 2: \(w = 3.2 - 0.1 \times 6.4 = 2.56\)
Step 3: \(w = 2.56 - 0.1 \times 5.12 = 2.048\)

Each step reduces \(w\) by 20%, converging geometrically toward \(w^* = 0\).

Learning Rate Behaviour

Learning RateBehaviour
Too small (1e-4)Converges but very slowly
Well chosen (0.01)Smooth convergence
Too large (10)Overshoots — loss oscillates or diverges

Variants

  • Batch GD: all \(n\) samples per step — stable, slow
  • SGD: 1 sample per step — fast, noisy, can escape local minima
  • Mini-batch GD: 32–512 samples — the deep learning standard
  • Adam / RMSprop: adaptive per-parameter learning rates

Section 3 — Partial Derivatives & The Gradient ∂f/∂wᵢ · ∇L · Steepest ascent

Real models have millions of parameters. Partial derivatives let us differentiate w.r.t. each weight independently while treating all others as constants.

The Gradient Vector
\[ \nabla L = \begin{bmatrix} \partial L/\partial w_1 \\ \vdots \\ \partial L/\partial w_n \end{bmatrix} \] Points in the direction of steepest ascent. Gradient descent: \(\mathbf{w} \leftarrow \mathbf{w} - \alpha \nabla L\).
Worked Example
\(f(w_1, w_2) = w_1^2 + 3w_1 w_2\).

\(\partial f/\partial w_1 = 2w_1 + 3w_2\) (treat \(w_2\) as constant)
\(\partial f/\partial w_2 = 3w_1\) (treat \(w_1\) as constant)

\(\nabla f = [2w_1 + 3w_2,\; 3w_1]^\top\)

Section 4 — The Chain Rule Composite functions · Backpropagation

Chain Rule
If \(y = f(g(x))\): \[ \frac{dy}{dx} = f'(g(x)) \cdot g'(x) \] "Outer derivative evaluated at the inner, times derivative of the inner."
Worked Example
\(\frac{d}{dx}\left[e^{3x^2}\right]\): outer \(e^u \to e^u\), inner \(3x^2 \to 6x\).

Result: \(6x\,e^{3x^2}\)
Sigmoid Derivative
\(\sigma(z) = 1/(1+e^{-z})\),  \(\sigma'(z) = \sigma(z)\left(1-\sigma(z)\right)\)

At \(z=0\): \(\sigma(0)=0.5\), \(\sigma'(0)=0.25\) — the maximum slope. For large \(|z|\), \(\sigma'\to 0\) → vanishing gradient.
Backpropagation = Chain Rule
For loss \(L\) through 3 layers: \[ \frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z_3} \cdot \frac{\partial z_3}{\partial a_2} \cdot \frac{\partial a_2}{\partial z_2} \cdot \frac{\partial z_2}{\partial W_1} \] PyTorch, JAX, and TensorFlow compute this automatically via autograd.

Section 5 — Loss Functions MSE · Cross-entropy · Statistical foundations

Mean Squared Error (Regression)
\[ L = \frac{1}{n} \sum_{i=1}^n (\hat{y}_i - y_i)^2 \] Differentiable everywhere. Penalises outliers heavily (squaring). MLE under Gaussian noise.
Binary Cross-Entropy (Classification)
\[ L = -\frac{1}{n}\sum_{i=1}^n \left[ y_i \log \hat{y}_i + (1-y_i)\log(1-\hat{y}_i) \right] \] Used when \(y \in \{0,1\}\). MLE under Bernoulli. Log penalises confident wrong predictions heavily.
LossUse caseKey property
MSERegressionDifferentiable; sensitive to outliers
MAERegressionRobust; not differentiable at 0
Binary cross-entropyBinary classificationProbabilistic; log-based
Categorical cross-entropyMulti-classUsed with softmax output

Section 6 — Regression as Optimisation Normal equations · Ridge · Gradient of MSE

Gradient of MSE
\[ \nabla_\mathbf{w} L = \frac{2}{n} \mathbf{X}^\top (\mathbf{X}\mathbf{w} - \mathbf{y}) \] Setting to zero → Normal Equations: \[ \mathbf{w}^* = (\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y} \] This is the projection of \(\mathbf{y}\) onto the column space of \(\mathbf{X}\) — exactly Week 2's least-squares projection.
Ridge Regression (L2 Regularisation)
\[ L(\mathbf{w}) = \frac{1}{n}\|\mathbf{X}\mathbf{w}-\mathbf{y}\|^2 + \lambda\|\mathbf{w}\|^2 \] \[ \mathbf{w}^* = (\mathbf{X}^\top\mathbf{X} + \lambda\mathbf{I})^{-1}\mathbf{X}^\top\mathbf{y} \] \(\lambda\mathbf{I}\) guarantees invertibility and shrinks weights — preventing overfitting.

Forward Pass

\[ a^{(0)} = \mathbf{x},\quad z^{(l)} = W^{(l)} a^{(l-1)} + b^{(l)},\quad a^{(l)} = \sigma(z^{(l)}) \]
Why Nonlinearity is Essential
\(W_2(W_1\mathbf{x}+b_1)+b_2 = W_\text{eff}\mathbf{x}+b_\text{eff}\). Without activations, any depth = one linear layer. Nonlinearities make depth genuinely powerful.

Section 8 — Python for Calculus & Optimisation NumPy · scipy · Autograd

Gradient Descent in NumPy

import numpy as np # Minimise L(w) = (w - 3)^2 w, lr = np.array([0.0]), 0.1 for _ in range(50): grad = 2 * (w - 3) w = w - lr * grad print(w) # ≈ [3.0]

scipy.optimize

from scipy.optimize import minimize res = minimize(lambda w: (w[0]-3)**2, x0=[0.0]) print(res.x) # [3.0]

Automatic Differentiation (PyTorch)

import torch w = torch.tensor([5.0], requires_grad=True) L = (w - 3)**2 L.backward() # fills w.grad with 2*(5-3)=4 with torch.no_grad(): w -= 0.1 * w.grad # w = 4.6
