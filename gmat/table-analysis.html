Lesson 1

Sorting & Filtering Data

Objectives
  • Use the sortable table interface to reorder rows by any column.
  • Identify the correct row(s) that satisfy a given filter condition.
  • Work efficiently by sorting by the most relevant column before scanning.
📌 Core Concept

Table Analysis questions present a sortable spreadsheet. You must answer Yes/No or select a value by locating the relevant row(s). The key skill is efficient navigation — sort by the column most relevant to the question, then scan for values meeting the stated criteria.

Most test-takers waste time reading every cell — instead, read the question first and let it direct your attention to the relevant column or row range.

How to Approach It
  1. 1Read the question to identify the key column(s) involved before examining the table.
  2. 2Mentally sort by the most restrictive criterion first to reduce the candidate rows.
  3. 3Scan for rows that satisfy ALL conditions simultaneously.
  4. 4Select Yes/No or provide the value for the qualifying row(s).
Example

The table below shows data for five retail stores. It can be sorted by any column.

StoreRegionRevenue ($M)Employees
HarlowWest62210
KingsleyEast48180
OakfieldWest85340
RedmoorNorth3195
TivertonWest57190

Which West-region store has the second-highest revenue?

(A) Harlow(B) Kingsley(C) Oakfield(D) Tiverton(E) Redmoor
▶ Show Answer

Answer: (A). Filter to West region first: Harlow ($62M), Oakfield ($85M), Tiverton ($57M). Sorted by revenue: Oakfield ($85M) #1, Harlow ($62M) #2, Tiverton ($57M) #3. Common trap: reading from the unsorted original table and picking Oakfield as the answer to the "second-highest" question.

Common Traps
  • Treating the table as pre-sorted when it is not — always confirm the sort order before drawing conclusions about the "top" or "bottom" rows.
  • Reading the entire table instead of filtering immediately — target only the column the question references.
Summary
  • Sort by the column most relevant to the question first to immediately reduce candidates.
  • Filter conditions apply simultaneously — AND logic, not OR logic.
  • Speed matters — efficient column targeting is the core skill in Table Analysis.
💡 Quick Tip

On the real GMAT, you can click a column header to sort. Practice identifying which column to sort first — it eliminates most rows instantly and saves valuable time.

Lesson 2

Multi-Condition Questions

Objectives
  • Translate a multi-part question into a set of simultaneous filter conditions.
  • Count or identify rows that satisfy all conditions — AND logic, not OR logic.
  • Distinguish between "at least one condition" (OR) and "all conditions" (AND) question phrasing.
📌 Core Concept

Multi-condition Table Analysis questions ask you to find rows meeting two or more criteria at once. These are AND problems — a row qualifies only if every listed condition is true for that row.

Misreading one condition as optional (OR logic) is the most common source of error. A structured approach — apply each filter sequentially to a shrinking subset — eliminates this mistake.

How to Approach It
  1. 1List every condition from the question stem before touching the table.
  2. 2Sort by the first condition to isolate a subset of rows.
  3. 3Within that subset, apply remaining conditions to further narrow down.
  4. 4Count or report on the final qualifying set.
Example

Using the same table from Lesson 1 (Store | Region | Revenue $M | Employees).

How many stores are in the West region AND have revenue greater than $60M AND have more than 200 employees?

(A) 0(B) 1(C) 2(D) 3(E) 4
▶ Show Answer

Answer: (C). Apply all three conditions simultaneously. West region stores: Harlow (62, 210), Oakfield (85, 340), Tiverton (57, 190). Condition: Revenue > $60M AND Employees > 200. Harlow: 62 > 60 ✓, 210 > 200 ✓. Oakfield: 85 > 60 ✓, 340 > 200 ✓. Tiverton: 57 > 60 ✗ (fails). Answer = 2. Common trap: counting all 3 West-region stores, missing the Revenue filter on Tiverton.

Common Traps
  • Counting rows that satisfy any one condition instead of all conditions simultaneously — these questions require AND logic throughout.
  • Using the wrong column — check column labels carefully as table headers can be long and similar-sounding.
Summary
  • Multi-condition = AND logic. Every condition must be satisfied by each qualifying row.
  • Sort by the first condition, then filter within the reduced set for subsequent conditions.
  • Re-read each condition after finding your answer to verify you have not accidentally dropped one.
💡 Quick Tip

Write the conditions down in shorthand (e.g., "Revenue > 10M AND Region = West") before scanning. This prevents accidentally dropping a condition mid-scan under time pressure.
